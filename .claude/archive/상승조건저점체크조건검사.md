## 1. 현황 분석

### 1-1. chart_buy가 뜨기까지의 전체 필터 체인

```
_analyze_ticker()
  ├─ [A] _is_dead_cross(ema)         → EMA20 < EMA50이면 전체 스킵 ✅ 적용 중
  ├─ [B] _check_trend(df, ema)       → BULL/BEAR/NEUTRAL 계산만, chart_buy엔 미사용 ❌
  ├─ [C] _check_buy_signal_precise() → Pine Script 시뮬레이션 (BUY / SQZ BUY)
  │       ├─ RSI >= 80 AND squeeze=0 이면 스킵
  │       ├─ 마지막 봉이 7일 이상 오래됐으면 스킵
  │       └─ 10거래일 이내 BUY/SQZ BUY 마커 존재 여부
  └─ [D] _passes_volume_filter()     → 신호일 거래량 > 직전 5일 평균 × 1.5배 ✅ 적용 중
```

### 옵션 B — 완화된 중간 조건 (권장 후보)

EMA200 정렬 없이, **EMA20 > EMA50 + EMA20 기울기 양수** 로 상승 진입 초기도 포함

```python
def _is_uptrend_mild(ema: dict) -> bool:
    """완화된 상승 추세: EMA20 > EMA50 + EMA20 최근 기울기 양수."""
    e20 = ema.get("ema_20")
    e50 = ema.get("ema_50")
    if e20 is None or e50 is None:
        return False
    last_e20 = float(e20.iloc[-1])
    last_e50 = float(e50.iloc[-1])
    if last_e20 <= last_e50:
        return False
    # EMA20 최근 10봉 기울기
    recent = e20.dropna().tail(10)
    if len(recent) < 10:
        return False
    slope = float(recent.iloc[-1]) - float(recent.iloc[0])
    return slope > 0
```

**판정 기준:**
- EMA20 > EMA50 (골든크로스 이상)
- EMA20 최근 10봉 기울기 양수 (상승 추세 진행 중)

**장점:** 기존 dead cross 필터보다 한 단계 더 엄격하면서 EMA200 불필요  
**단점:** SQZ BUY 신호(횡보 돌파)에 기울기 조건이 맞지 않을 수 있음

## 3. 저점 상승 체크 설계 방안

### 3-1. 스윙 저점(Swing Low) 탐지 알고리즘

스윙 저점 = 앞뒤 N봉보다 낮은 로컬 최저점

```python
def _find_swing_lows(df: pd.DataFrame, window: int = 2) -> list[tuple[int, float]]:
    """
    스윙 저점 탐지.
    window: 전후로 비교할 봉 수 (window=2 → 2봉 앞뒤 모두 비교)
    반환: [(인덱스, 저점 가격), ...]
    """
    lows = df["low"].values
    swing_lows = []
    for i in range(window, len(lows) - window):
        is_low = all(lows[i] <= lows[i - j] for j in range(1, window + 1)) and \
                 all(lows[i] <= lows[i + j] for j in range(1, window + 1))
        if is_low:
            swing_lows.append((i, float(lows[i])))
    return swing_lows
```

**window=2의 의미:** 해당 봉이 좌우 2봉보다 낮아야 스윙 저점 인정  
→ 너무 작으면 노이즈, 너무 크면 스윙이 드물어짐  
→ 일봉 기준 window=2~3 적정

### 3-2. 저점 상승 추세 판정

```python
def _check_higher_lows(df: pd.DataFrame, lookback: int = 40, min_swings: int = 2) -> bool:
    """
    최근 lookback 봉 내에서 스윙 저점이 상승하고 있는지 확인.
    
    lookback: 탐색할 최근 봉 수 (일봉 기준 40봉 ≈ 2개월)
    min_swings: 판정에 필요한 최소 스윙 저점 수 (2개 이상 필요)
    반환: True이면 저점이 상승 중 (higher lows)
    """
    swing_lows = _find_swing_lows(df, window=2)
    
    # 최근 lookback 봉 이내의 스윙 로우만
    start_idx = max(0, len(df) - lookback)
    recent = [(i, v) for i, v in swing_lows if i >= start_idx]
    
    if len(recent) < min_swings:
        return False  # 스윙 저점 부족 → 판단 불가 → 통과 허용
    
    # 최근 2개 저점 비교 (마지막 저점 > 이전 저점)
    last_two = recent[-2:]
    return last_two[-1][1] > last_two[0][1]
```

### 3-3. 저점 판정 파라미터 가이드

| 파라미터 | 값 | 의미 |
|----------|-----|------|
| `window` | 2 | 앞뒤 2봉보다 낮아야 스윙 저점 인정 |
| `lookback` | 40 | 최근 40거래일(약 2개월) 내 탐색 |
| `min_swings` | 2 | 저점이 2개 이상 있어야 판정 |
| 부족 시 | True 반환 | 신규 상장·데이터 부족 시 필터 패스 |

### 4-1. 새 함수 2개 추가 — `full_market_scanner.py`

```python
def _ema20_slope_positive(e20_series, lookback: int = 10) -> bool:
    """EMA20 최근 lookback봉 기울기가 양수인지 확인."""
    if e20_series is None:
        return False
    recent = e20_series.dropna().tail(lookback)
    if len(recent) < lookback:
        return False
    return float(recent.iloc[-1]) > float(recent.iloc[0])


def _check_higher_lows(df: pd.DataFrame, lookback: int = 40, min_swings: int = 2) -> bool:
    """최근 lookback봉 내 스윙 저점이 상승 추세인지 확인. 저점 부족 시 True(필터 패스)."""
    lows = df["low"].values
    # 스윙 저점 탐지 (window=2)
    swing_lows = []
    for i in range(2, len(lows) - 2):
        if lows[i] <= lows[i-1] and lows[i] <= lows[i-2] and \
           lows[i] <= lows[i+1] and lows[i] <= lows[i+2]:
            swing_lows.append((i, float(lows[i])))

    start_idx = max(0, len(df) - lookback)
    recent = [(i, v) for i, v in swing_lows if i >= start_idx]

    if len(recent) < min_swings:
        return True  # 데이터 부족 → 필터 건너뜀

    last_two = recent[-2:]
    return last_two[-1][1] > last_two[0][1]  # 마지막 저점 > 이전 저점
```

## 8. 추가 고려사항

### 8-1. `unified_scanner.py`도 동일한 개선 필요
현재 `unified_scanner.py`에는 `e20 < e50`이면 스킵하는 dead cross 필터가 있으나,  
`full_market_scanner.py`와 동일하게 EMA20 기울기 + higher_lows 조건을 추가해야 일관성 유지

### 8-2. 파라미터 튜닝 고려
- `lookback=40`은 약 2개월치 일봉 — 너무 짧으면 노이즈, 너무 길면 상승 전환 초기 종목 탈락
- 초기 배포 후 false positive 비율 모니터링 후 조정 권장

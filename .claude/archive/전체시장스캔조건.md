# 전체 시장 스캔 조건 정리

## 아키텍처 개요

두 가지 스캔 시스템이 병렬로 운영됨:

| 시스템 | API | 대상 | 방식 | 용도 |
|--------|-----|------|------|------|
| **전체 시장 스캔** (신규) | `/api/scan/full/*` | stock_master 전종목 3,700+ | DB 스냅샷 (백그라운드) | 전종목 스크리닝 |
| 통합 스캔 (기존) | `/api/scan/unified` | 하드코딩 ~160종목 | 메모리 캐시 (실시간) | 대시보드 빠른 표시 |

## 전체 시장 스캔 (Full Market Scanner)

### 핵심 흐름

```
APScheduler (평일 매시 :30, KST 9:30~15:30)
  → stock_master DB에서 전종목 로드 (KOSPI 1,804 + KOSDAQ 1,785 + US 113 + 암호화폐 10)
  → 100개씩 38 청크 분할
  → yfinance 배치 다운로드 (청크당 ~10초, 타임아웃 120초)
  → 지표 계산 + 신호 판정
  → scan_snapshot / scan_snapshot_item DB 저장
  → 약 7분 소요
```

### API 엔드포인트

| 엔드포인트 | 메서드 | 설명 |
|-----------|--------|------|
| `/api/scan/full/latest` | GET | 최신 완료 스냅샷 즉시 반환 (~30ms) |
| `/api/scan/full/status` | GET | 스캔 진행 상태 + 진행률 |
| `/api/scan/full/trigger` | POST | 수동 스캔 트리거 (백그라운드 실행) |
| `/api/scan/full/history` | GET | 최근 10개 스냅샷 이력 |

### 공통 설정

| 항목 | 값 |
|------|-----|
| 데이터 기간 | 1년 (1y) |
| 봉 간격 | 일봉 (1d) |
| 최소 캔들 수 | 60개 (SCAN_MIN_CANDLES) |
| 데이터 소스 | yfinance 배치 다운로드 |
| 청크 크기 | 100개 (CHUNK_SIZE) |
| 청크 타임아웃 | 120초 |
| 청크 간 대기 | 1초 (rate limit 방지) |
| 스냅샷 보관 | 최근 10개 (MAX_SNAPSHOTS) |

## 1. 전체 시장 스캔 — 추천 (picks)

### 진입 조건

```
squeeze_level >= 2  (MID SQ 이상)
AND trend == "BULL"
AND EMA20 >= EMA50  (Dead Cross 아님)
```

### 트렌드 필터

```
BULL: EMA20 > EMA50 > EMA200 AND 가격 > EMA20 AND EMA20 기울기(5봉) > 0
BEAR: EMA20 < EMA50 < EMA200 AND 가격 < EMA20
NEUTRAL: 그 외
```

### 스퀴즈 레벨

| 레벨 | 라벨 | 상태 |
|------|------|------|
| 0 | NO SQ | BB 확장 (스퀴즈 아님) |
| 1 | LOW SQ | 넓은 스퀴즈 |
| 2 | MID SQ | 보통 스퀴즈 |
| 3 | MAX SQ | 극한 압축 |

- `pandas_ta.squeeze_pro()` 4단계 판정

### 신뢰도 점수 (0~100)

| 조건 | 점수 |
|------|------|
| 스퀴즈 레벨 × 25 | 최대 75 |
| BULL 트렌드 | +15 |
| RSI < 40 (과매도) | +10 |
| BB %B < 0.3 (하단 근접) | +5 |
| MACD > 0 (양수) | +5 |
| 거래량 비율 > 1.0 (평균 이상) | +5 |

### 정렬

- confidence 내림차순

---

## 2. MAX SQ 폭발 임박 (max_sq)

### 진입 조건

```
squeeze_level >= 3   # MAX SQ (극한 압축)
AND trend == "BULL"  # EMA 정배열
AND EMA20 >= EMA50   # Dead Cross 아님
```

### 의미

- BB가 극도로 좁아진 상태 → 방향성 폭발 직전
- BULL 트렌드 확인으로 상방 돌파 가능성 높음

---

## 3. 차트 BUY 신호 (chart_buy)

### 경량 판정 로직 (전종목 스캔용)

전종목 스캔에서는 `_simulate_signals` 대신 경량화된 `_check_buy_signal_light` 사용.
최근 3일 내 조건을 직접 체크.

#### Regular BUY

```
close <= lower_band AND RSI < 40
OR
전봉 close < lower_band AND 현봉 close > lower_band AND RSI < 40 (반전)
```

#### SQZ BUY (스퀴즈 해소)

```
현재 squeeze=0 AND 이전 squeeze>0 (해소)
AND MACD histogram > 0 (양수 모멘텀)
```

### 사용 지표

| 지표 | 파라미터 |
|------|----------|
| Bollinger Bands | period=20, std=2.0 |
| RSI | period=14 |
| MACD | fast=12, slow=26, signal=9 |
| EMA | 20, 50, 200 (트렌드 판정) |
| Squeeze Pro | pandas_ta squeeze_pro |

## 프론트엔드 UX

1. **첫 접속**: DB 스냅샷 즉시 표시 (~30ms) / 없으면 "스캔 이력 없음"
2. **설정 화면**: 최근 10개 스냅샷 이력 테이블 + 수동 스캔 버튼
3. **스캔 진행 중**: 프로그레스바 (5초 폴링) + 이전 결과 유지
4. **스캔 완료**: 자동 이력 갱신

## Signal Engine (Watchlist 개별 종목 모니터링)

Watchlist에 등록된 종목에 대해 10분마다 APScheduler가 실행하는 별도 신호 엔진.

### 민감도 설정

| 설정 | 필요 조건 수 | RSI 매수 | RSI 매도 | BB 매수 | BB 매도 | 거래량 배수 | 기본 점수 |
|------|-------------|---------|---------|--------|--------|-----------|---------|
| strict (엄격) | 4개 | <30 | >70 | <0.05 | >0.95 | ×1.2 | 80 |
| normal (보통) | 3개 | <35 | >65 | <0.15 | >0.85 | ×1.1 | 70 |
| sensitive (민감) | 2개 | <40 | >60 | <0.25 | >0.75 | ×1.0 | 60 |

### BUY 조건 (4개 중 N개 충족)

1. BB %B ≤ 임계값 (하단 근접)
2. RSI < 임계값 (과매도)
3. MACD 히스토그램 > 이전값 (모멘텀 전환)
4. 거래량 > 임계값 × 20일 평균 (거래량 확인)

### 신뢰도 등급

| 등급 | 점수 |
|------|------|
| STRONG | 90+ |
| NORMAL | 70+ |
| WEAK | 60+ |
| 없음 | 60 미만 |

## 1. 기존 기능 중 디자인 파일에 누락된 내역

디자인 파일이 생성될 당시 존재하지 않았거나 반영되지 않은 기능.  
구현 시 **반드시 동작을 유지**해야 하며, 새 디자인 레이아웃에 맞게 통합해야 한다.

| # | 기능 | 현재 위치 | 처리 방향 |
|---|------|----------|----------|
| 1 | **섹션 chevron 토글** (관심/추천/눌림목/대형주 접기·펼치기) | `Dashboard.tsx` — `closedSec` state | 새 레이아웃에도 토글 유지 |
| 2 | **대형주 섹션** (pullback 필터 + is_large_cap) | `Dashboard.tsx` + `full_market_scanner.py` | 디자인 ScanStatusPanel에 숫자 슬롯은 있음 — 목록 섹션 통합 필요 |
| 3 | **요약 카운트 헤더** (추천 N개 / 눌림목 N개 / 대형주 N개) | `Dashboard.tsx` MarketScanBox | ScanStatusPanel 4칸으로 대체 — 기존 인라인 카운트 제거 가능 |
| 4 | **관심종목 종목 검색·추가** (검색창 + 드롭다운) | `Dashboard.tsx` 상단 검색 UI | WatchlistPanel에 추가 버튼/검색 흐름 통합 필요 |
| 5 | **관심종목 삭제** (카드별 휴지통 아이콘) | `SignalCard.tsx` | WatchlistPanel 카드에 삭제 인터랙션 유지 필요 |
| 6 | **모바일 스냅 스크롤** (5섹션 세로 스냅) | `Dashboard.tsx` snapRef + usePageSwipe | 디자인은 일반 스크롤 — 스냅 유지 여부 결정 필요 |
| 7 | **새탭 열기 클릭** (SignalCard → 종목 상세 새탭) | `SignalCard.tsx` `window.open(_blank)` | 새 SignalCard 디자인에도 클릭 동작 유지 |
| 8 | **SentimentPanel** (공포탐욕 기존 패널) | `SentimentPanel.tsx` | FearGreedPanel로 교체 — 기존 API 호출 유지 |
| 9 | **조회조건 페이지** 파이프라인 텍스트 표 | `ScanConditions.tsx` | pc-pipeline.jsx SVG 플로우차트로 교체 시 내용 보존 |
| 10 | **눌림목 섹션 목록** | `Dashboard.tsx` pullbackItems | SectorSignals 또는 별도 섹션으로 통합 필요 |

## 2. 디자인 파일에 있으나 아직 미구현인 기능

디자인 파일에 UI가 정의되어 있지만 현재 앱에 없는 기능.  
`(더미)` 표시 = 실데이터 연동 없이 UI 형태만 먼저 구현 가능.

### 2-A. PC 레이아웃

| # | 기능 | 디자인 파일 | 비고 |
|---|------|------------|------|
| 1 | **PC TopNav 터미널 스타일** — ⌘K SEARCH 버튼, 탭 뱃지, 로고 | `pc-dashboard.jsx` `PcTopNav` | 탭: 마켓·스캔·추천종목·기록·파이프라인 |
| 2 | **Market Ticker 8종목 스파크라인 바** — KOSPI·KOSDAQ·S&P500·NASDAQ·VIX·USD/KRW·BTC·US10Y | `pc-dashboard.jsx` `MarketTicker` | US10Y 데이터 없으면 `—` 표시 (더미 가능) |

| 4 | **ScanStatusPanel 4칸 터미널 스타일** — 추천/눌림목/대형주/데드크로스 큰 숫자 | `pc-dashboard.jsx` `ScanStatusPanel` | 스캔 진행률 바 + REFRESH 버튼 포함 |
| 5 | **WatchlistPanel KR/US 그리드** — 태그 뱃지 포함 | `pc-dashboard.jsx` `WatchlistPanel` | 관심종목 추가·삭제 인터랙션 통합 필요 |
| 6 | **SectorSignals 패널** — 섹터별 신호 목록 + EMA 분포 바 + 시장 필터 탭 | `pc-dashboard.jsx` `SectorSignals` | 백엔드 섹터 분류 API 필요 (현재 없음) |

| 9 | **SqueezeStages** — 스퀴즈 단계 시각화 (진입전·스퀴즈·해소·모멘텀) | `pc-detail.jsx` `SqueezeStages` | 더미 UI 가능 |

## 3. 백엔드 신규 API가 필요한 항목

아래 항목은 현재 백엔드 API가 없어 프론트 UI만 먼저 구현하고, 별도 스펙에서 연동한다.

| 항목 | 필요한 데이터 |
|------|-------------|
| Market Ticker (실시간) | KOSPI·KOSDAQ·S&P500·NASDAQ·VIX·USD/KRW·BTC·US10Y 현재가 + 등락률 + 30봉 히스토리 |
| SectorSignals | 종목별 섹터 분류 + 신호 집계 |
| BUY 사례 스크랩 KPI | 거래 기록(매수일·매도일·수익률) 저장 및 조회 |
| 스캔 히스토리 로그 | 스캔 실행 시각·대상 종목 수·BUY 수·소요시간 |
| EntryPlanPanel | (선택) 매수가 제안 로직 |

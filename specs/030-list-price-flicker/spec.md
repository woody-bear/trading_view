## Clarifications

### Session 2026-04-23

- Q: 목록 화면 가격 갱신 방식은 종목상세(1초)와 동일한가, 아니면 다른 주기를 사용하는가? → A: 목록 화면 전체를 3~5초마다 일괄 갱신 (개별 종목별 1초 연결은 동시 다수 종목 시 부하 과다)
- Q: 가격 갱신 시 등락률(%)도 함께 깜빡이며 업데이트되어야 하는가? → A: 현재가 + 등락률 둘 다 갱신 및 깜빡임
- Q: 등락률이 음수(-)일 때 표시 색상은 무엇인가? → A: 파란색(blue). 하락 시 깜빡임 색상도 파란색으로 적용 (양수=초록, 음수=파란색)

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 목록 화면 실시간 가격 깜빡임 (Priority: P1)

사용자가 관심종목·추천종목·눌림목·대형주 중 하나를 보고 있을 때, 해당 종목의 주가와 등락률이 3~5초마다 갱신되며 값이 변경되면 잠깐 깜빡이며(flash) 새 값으로 바뀐다. 가격이 오르면 초록색, 내리면 파란색으로 깜빡인다. 등락률은 평소에도 양수=초록색, 음수=파란색으로 표시된다.

**Why this priority**: 핵심 UX 개선 — 사용자가 목록을 이탈하지 않고도 가격 변동을 즉각 인지할 수 있어 모니터링 효율이 올라간다.

**Independent Test**: 대시보드의 관심종목 목록만 열고 3~5초 대기 후 가격 갱신이 발생했을 때, 해당 종목 행의 현재가와 등락률이 올바른 색상으로 깜빡이며 변경되면 검증 완료.

**Acceptance Scenarios**:

1. **Given** 사용자가 관심종목 목록을 보고 있고, **When** 3~5초 주기 갱신 시 종목 A의 가격이 이전보다 높아지면, **Then** 종목 A의 현재가와 등락률이 초록색으로 약 0.8초간 깜빡이며 새 값으로 변경된다.
2. **Given** 사용자가 관심종목 목록을 보고 있고, **When** 3~5초 주기 갱신 시 종목 A의 가격이 이전보다 낮아지면, **Then** 종목 A의 현재가와 등락률이 파란색으로 약 0.8초간 깜빡이며 새 값으로 변경된다.
3. **Given** 사용자가 추천종목·눌림목·대형주 목록을 보고 있고, **When** 목록 내 종목 가격이 갱신 주기에 변경되면, **Then** 동일한 색상 규칙(상승=초록, 하락=파란)으로 깜빡이며 현재가와 등락률이 업데이트된다.
4. **Given** 갱신 주기가 도래했으나 가격 변동이 없는 경우, **When** 데이터가 수신되어도, **Then** 깜빡임이 발생하지 않고 현재가와 등락률 표시는 그대로 유지된다.
5. **Given** 목록을 처음 로드했을 때, **When** 종목의 등락률이 음수이면, **Then** 깜빡임 없이 파란색으로 표시된다. 양수이면 초록색으로 표시된다.

### User Story 2 - 화면 전환 후 연결 재개 (Priority: P2)

사용자가 목록 화면에서 다른 탭으로 이동했다가 돌아왔을 때, 실시간 가격 업데이트가 자동으로 재개된다.

**Why this priority**: 목록 화면을 나갔다 돌아올 때 가격이 멈춰 있으면 신뢰도 문제 발생.

**Independent Test**: 대시보드 → 다른 페이지 이동 → 대시보드 복귀 후 가격 깜빡임이 다시 동작하면 검증 완료.

**Acceptance Scenarios**:

1. **Given** 사용자가 목록 화면에서 다른 페이지로 이동하고, **When** 다시 목록 화면으로 돌아오면, **Then** 실시간 가격 업데이트가 자동으로 재개된다.
2. **Given** 실시간 가격 연결이 끊어진 경우, **When** 화면이 활성화되면, **Then** 자동으로 재연결을 시도한다.

---

### Edge Cases

- 목록에 50개 이상 종목이 동시에 표시될 때 3~5초 일괄 갱신이 발생해도 UI 반응 속도가 저하되지 않는가?
- 가격 데이터가 없거나 수신 실패 시, 기존에 표시된 현재가와 등락률이 그대로 유지되는가(빈 값으로 바뀌지 않는가)?
- 사용자가 목록 화면을 보지 않는 동안(다른 탭 포커스) 불필요한 연결이 유지되지 않는가?
- 등락률이 0.00%인 경우(보합) 색상은 어떻게 표시되는가? (기본: 흰색/회색으로 표시)

---

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 관심종목·추천종목·눌림목·대형주 목록 화면에서 각 종목의 현재가와 등락률이 3~5초 주기로 일괄 갱신되어야 한다.
- **FR-002**: 현재가가 이전 값보다 높아지면 현재가와 등락률 표시 영역이 초록색으로 약 0.8초간 깜빡여야 한다.
- **FR-003**: 현재가가 이전 값보다 낮아지면 현재가와 등락률 표시 영역이 파란색으로 약 0.8초간 깜빡여야 한다.
- **FR-004**: 가격 변동이 없는 경우 깜빡임이 발생해서는 안 된다.
- **FR-005**: 깜빡임 시각 효과(애니메이션 방식, 지속 시간)는 종목상세 화면(SignalDetail)과 동일하게 적용하되, 색상 규칙은 상승=초록, 하락=파란색으로 이 기능에서 별도 정의한다.
- **FR-006**: 목록 화면에서 이탈 시 실시간 연결이 해제되어 불필요한 리소스를 소비하지 않아야 한다.
- **FR-007**: 가격 수신 실패 또는 데이터 부재 시 마지막으로 수신된 현재가와 등락률이 그대로 표시되어야 한다(빈 값으로 바뀌지 않음).
- **FR-008**: 등락률은 항상 값의 부호에 따라 색상이 결정된다 — 양수(+)이면 초록색, 음수(-)이면 파란색, 보합(0.00%)이면 기본 텍스트 색상(흰색/회색)으로 표시한다. 이 색상은 깜빡임 여부와 무관하게 상시 적용된다.

### Key Entities

- **종목 행(Stock Row)**: 목록 내 개별 종목을 나타내는 UI 단위. 심볼·이름·현재가·등락률 등을 포함하며, 현재가와 등락률이 가격 업데이트 대상이 됨.
- **가격 플래시(Price Flash)**: 현재가 변동 시 트리거되는 시각 효과. 상승=초록, 하락=파란색으로 현재가와 등락률에 동시 적용되며 약 0.8초 후 자동 소멸.
- **등락률 색상(Change Rate Color)**: 등락률 수치의 부호에 따라 상시 적용되는 텍스트 색상 규칙. 양수=초록, 음수=파란색, 보합=기본색.
- **실시간 가격 피드(Price Feed)**: 서버에서 클라이언트로 3~5초 주기로 전달되는 현재가·등락률 일괄 데이터.

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 관심종목·추천종목·눌림목·대형주 목록 화면 모두에서 현재가와 등락률의 가격 깜빡임이 동작한다 (4개 화면 100% 적용).
- **SC-002**: 가격 갱신 주기(3~5초)마다 목록 화면의 현재가와 등락률 표시가 업데이트된다.
- **SC-003**: 깜빡임 색상 규칙이 올바르게 적용된다 (상승=초록, 하락=파란색). 등락률 정적 색상도 동일 규칙(양수=초록, 음수=파란색)을 따른다.
- **SC-004**: 목록 화면에 50개 종목이 동시에 표시되는 상황에서도 화면 반응 속도가 눈에 띄게 저하되지 않는다.
- **SC-005**: 목록 화면에서 다른 페이지로 이동 후 돌아왔을 때 가격 업데이트가 자동 재개된다.

---

## Assumptions

- 목록 화면의 가격 일괄 갱신은 화면에 표시된 종목 심볼 목록을 서버에 전달해 한 번에 현재가·등락률을 응답받는 방식으로 구현한다.
- 목록 화면의 각 종목 행은 심볼과 시장(market) 정보를 이미 보유하고 있다고 가정한다.
- 가격 갱신 주기는 3~5초로 하며, 정확한 값은 구현 시 성능 테스트 결과에 따라 결정한다.
- 기존 스타일(색상 변수, 폰트) 체계를 그대로 활용하며 별도 디자인 변경은 없다.
- 깜빡임 애니메이션 방식은 기존 `usePriceFlash` 훅을 참고하되, 하락 색상을 파란색으로 변경하여 적용한다.
- 파란색(blue)은 한국 증권 앱의 관례(하락=파란색)를 따른다.

# Feature Specification: 메인페이지 종목 카드 항목 및 디자인 개선

**Feature Branch**: `011-stock-card-redesign`
**Created**: 2026-04-01
**Status**: Draft
**Input**: 메인페이지에 표시되는 종목카드에 항목과 디자인을 개선

## Clarifications

### Session 2026-04-01

- Q: "특징" 라벨의 범위 → A: 지표 조건 + 신호 강도 + 시장 유형 배지 + 섹터/업종 모두 포함
- Q: 전체 사이트 배경색 조정 범위 → A: 페이지 배경 slate-900→slate-800, 카드 배경 slate-800→slate-700 한 단계씩 밝게, 눈부심 없는 수준 유지
- Q: 라벨 개수 제한 → A: 시장 유형·섹터는 항상 표시(고정), 지표 조건 라벨은 중요도 순 최대 4개
- Q: 섹터 데이터 출처 → A: 국내는 stock_master의 섹터 필드, 미국은 시장 유형 배지(SP500/NQ100/R1000)로 대체, 외부 API 연동 없음

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 종목 카드에서 지표 상태 즉시 파악 (Priority: P1)

사용자가 메인페이지의 관심종목, 차트 BUY 신호, 투자과열, 추천 종목 카드를 보고 해당 종목의 현재 지표 상태와 신호를 한눈에 파악할 수 있다. 지표값 숫자만 나열되는 현재 방식에서, 의미 있는 상태를 시각적으로 강조하는 색상 라벨로 개선한다.

**Why this priority**: 종목 카드는 사용자가 가장 많이 보는 화면이며, 추가 클릭 없이 빠른 판단이 가능해야 한다.

**Independent Test**: 메인페이지에서 종목 카드를 보고, 추가 클릭 없이 해당 종목이 매수 조건에 해당하는지 5초 이내에 판단할 수 있다.

**Acceptance Scenarios**:

1. **Given** 관심종목에 BUY 신호 종목이 있을 때, **When** 메인페이지를 열면, **Then** 카드에 BUY 라벨이 녹색으로 표시된다.
2. **Given** RSI가 30 미만인 종목이 있을 때, **When** 관심종목 카드를 보면, **Then** "RSI 과매도" 라벨이 에메랄드 색으로 표시된다.
3. **Given** 스퀴즈 레벨 3인 종목이 있을 때, **When** 차트 BUY 신호 카드를 보면, **Then** "MAX SQ" 라벨이 빨간색으로 표시된다.
4. **Given** 투자과열 신호 종목이 있을 때, **When** 카드를 보면, **Then** RSI 과매수, 거래량 급증, BB 상단 등 과열 원인 라벨이 카드 하단에 표시된다.
5. **Given** 지표 데이터가 없는 종목이 있을 때, **When** 카드를 보면, **Then** 라벨 영역이 표시되지 않고 카드 레이아웃이 정상이다.

---

### User Story 2 - 4개 섹션 카드 간 일관된 색상 체계 (Priority: P2)

관심종목, 차트 BUY 신호, 투자과열, 추천 종목 4개 섹션의 카드가 동일한 색상 기준을 공유한다. 색상 체계(초록=매수 조건, 빨강=과열/매도, 파랑=중립 가능성, 보라=거래량 이상)가 모든 카드에 일관되게 적용된다.

**Why this priority**: 일관된 색상 언어가 학습되면 사용자가 어느 섹션을 보더라도 빠르게 정보를 읽을 수 있다.

**Independent Test**: 관심종목 카드의 "RSI 과매도" 라벨 색상과 차트 BUY 신호 카드의 동일 조건 라벨 색상이 동일하다.

**Acceptance Scenarios**:

1. **Given** 동일한 지표 조건(RSI < 30)이 있을 때, **When** 서로 다른 섹션의 카드를 비교하면, **Then** 동일한 색상 라벨이 표시된다.
2. **Given** 어떤 섹션의 카드든, **When** BUY 관련 라벨을 보면, **Then** 녹색 계열로 표시된다.
3. **Given** 어떤 섹션의 카드든, **When** 거래량 관련 라벨을 보면, **Then** 보라색 계열로 표시된다.

---

### User Story 3 - 전체 사이트 컬러 테마 밝기 조정 (Priority: P2)

현재 사이트의 전체 배경색이 너무 어두워 종목 카드가 배경에 묻혀 잘 보이지 않는다. 전체 배경과 카드 배경을 한 단계 밝은 mid-dark 톤으로 조정하여 카드와 배경 간 대비를 높이되, 눈부심이 없는 수준을 유지한다.

**Why this priority**: 배경 대비가 개선되면 라벨 추가 효과가 극대화되고, 장시간 모니터링 시 피로도가 줄어든다.

**Independent Test**: 변경 후 사이트를 어두운 환경에서 열었을 때 눈부심 없이 카드 내용이 기존보다 뚜렷하게 보인다.

**Acceptance Scenarios**:

1. **Given** 사이트를 열었을 때, **When** 메인페이지를 보면, **Then** 카드가 페이지 배경보다 명확하게 밝은 톤으로 구분되어 보인다.
2. **Given** 어두운 환경에서 화면을 볼 때, **When** 페이지를 보면, **Then** 눈부심 없이 자연스러운 다크 테마를 유지한다.
3. **Given** 텍스트 가독성 측면에서, **When** 카드 내 흰색/회색 텍스트를 보면, **Then** 변경 전보다 배경과의 대비가 높아 읽기 쉽다.

---

### User Story 4 - 카드 레이아웃 반응형 정합성 (Priority: P3)

라벨이 추가되어도 모바일과 PC 모두에서 카드가 깨지지 않고, 라벨이 여러 개일 때 자동으로 줄바꿈된다.

**Why this priority**: 라벨 개수가 많아질 경우 카드 레이아웃이 깨질 수 있어 안정성이 필요하다.

**Independent Test**: 라벨이 5개 이상인 종목 카드를 모바일(375px)과 PC(1280px)에서 확인하면 카드 밖으로 넘치지 않는다.

**Acceptance Scenarios**:

1. **Given** 라벨이 5개 이상인 경우, **When** 카드를 보면, **Then** 라벨들이 카드 안에서 줄바꿈 처리된다.
2. **Given** 모바일 화면(375px), **When** 카드를 보면, **Then** 핵심 정보(종목명, 가격, 신호)가 잘리지 않고 표시된다.

---

### Edge Cases

- 지표 데이터가 null/undefined인 경우: 해당 라벨만 표시하지 않고, 카드 레이아웃은 유지된다.
- 라벨이 0개인 경우: 라벨 영역 자체가 렌더링되지 않아 빈 공간이 생기지 않는다.
- 라벨이 6개 이상인 경우: 카드 너비 안에서 줄바꿈되며 카드 높이가 자연스럽게 늘어난다.
- 실시간 가격 업데이트로 카드가 깜빡일 때 라벨이 사라지거나 위치가 변하지 않는다.
- 종목명이 긴 경우(15자 이상) 종목명이 라벨 영역을 침범하지 않는다.

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 관심종목 카드는 signal_state(BUY/SELL)와 신호 강도(STRONG BUY / WEAK BUY / STRONG SELL)를 색상 라벨로 표시해야 한다.
- **FR-002**: RSI 조건(과매도 <30, 낮음 30~45, 과매수 >70)을 조건별 다른 색상 라벨로 표시해야 한다.
- **FR-003**: BB %B 조건(하단 <20%, 상단 >80%)을 라벨로 표시해야 한다.
- **FR-004**: 스퀴즈 레벨(LOW SQ Lv1, MID SQ Lv2, MAX SQ Lv3)을 레벨별 다른 색상 라벨로 표시해야 한다.
- **FR-005**: 거래량 조건(증가 ≥1.5x, 급증 ≥2x, 폭증 ≥3x)을 강도별 라벨로 표시해야 한다.
- **FR-006**: MACD 히스토그램이 양수일 때 상승 라벨(MACD↑)을 표시해야 한다.
- **FR-007**: 투자과열 카드에 과열 원인(RSI 과매수/극과매수, 거래량 급증/폭증, BB 상단/극상단)을 라벨로 표시해야 한다.
- **FR-008**: 추천 종목 카드에 RSI 과매도/낮음, BB 하단, 거래량 급증 라벨을 추가해야 한다.
- **FR-009**: 동일 지표 조건은 4개 섹션 모든 카드에서 동일한 색상 라벨로 표시되어야 한다.
- **FR-010**: 지표 데이터가 없는 경우 해당 라벨을 표시하지 않아야 한다.
- **FR-011**: 라벨이 없는 경우 라벨 영역 DOM을 렌더링하지 않아야 한다.
- **FR-012**: 라벨이 여러 개일 때 카드 너비 안에서 자동 줄바꿈되어야 한다.
- **FR-013**: 시장 유형 배지(KOSPI / KOSDAQ / SP500 / NASDAQ100 / RUSSELL1000 / CRYPTO 등)를 카드에 항상 표시해야 한다.
- **FR-014**: 국내 종목은 stock_master의 섹터 필드를 섹터 라벨로 표시한다. 미국 종목은 별도 섹터 조회 없이 시장 유형 배지(SP500 / NQ100 / R1000)로 대체 표시한다. 섹터 정보가 없는 종목은 해당 라벨을 생략한다.
- **FR-018**: 지표 조건 라벨은 중요도 순(신호 강도 > 스퀴즈 > RSI > BB > 거래량 > MACD) 최대 4개까지만 표시해야 한다.
- **FR-015**: 페이지 배경색을 slate-900 → slate-800 수준(한 단계)으로 밝게 조정해야 한다.
- **FR-016**: 카드 배경색을 slate-800 → slate-700 수준(한 단계)으로 밝게 조정하여 카드-배경 간 대비를 높여야 한다.
- **FR-017**: 조정된 색상은 어두운 환경에서도 눈부심이 없어야 하며, 카드 내 텍스트(흰색/회색 계열) 가독성이 기존 이상이어야 한다.

### 색상 체계 (지표 라벨 색상 기준)

- **초록 계열**: BUY, RSI 과매도, BB 하단, 상승추세, BB반전, 스퀴즈해소
- **빨강 계열**: SELL, RSI 과매수/극과매수, BB 극상단
- **주황/노랑 계열**: MAX SQ, MID SQ, LOW SQ, BB 상단
- **파랑/시안 계열**: RSI 낮음, 스퀴즈해소
- **보라 계열**: 거래량 증가, 거래량 급증, 거래량 폭증
- **틸 계열**: MACD↑

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 사용자가 종목 카드를 보고 신호 조건을 5초 이내에 파악할 수 있다.
- **SC-002**: 4개 섹션 모든 카드에서 동일 지표 조건은 동일 색상으로 100% 일관되게 표시된다.
- **SC-003**: 지표 데이터가 없는 종목 카드의 레이아웃 깨짐이 0건이다.
- **SC-004**: 모바일(375px)과 PC(1280px) 양쪽에서 카드 내 정보가 잘림이나 넘침 없이 표시된다.
- **SC-005**: 라벨 추가 후 카드 렌더링 속도가 기존 대비 체감상 차이가 없다.

## Non-Functional Requirements

- **NFR-001**: 컬러 테마 변경은 CSS 변수(Custom Properties) 수정으로 전체 페이지(메인, 스캔, 설정, 스크랩, BUY리스트 등)에 일괄 적용된다.
- **NFR-002**: 라벨 렌더링으로 인한 카드 초기 표시 시간 증가가 없어야 한다 (클라이언트 사이드 계산, 추가 API 호출 없음).

## Assumptions

- 현재 API 응답에 RSI, BB %B, squeeze_level, volume_ratio, macd_hist, signal_state 필드가 포함되어 있다.
- 투자과열 카드의 bb_pct_b, macd_hist 데이터가 API에서 제공된다고 가정하며, 없을 경우 해당 라벨은 표시하지 않는다.
- 라벨 표시 기준값(RSI 30/45/70, BB 20%/80%, 거래량 1.5x/2x/3x)은 현재 시스템 신호 기준과 동일하다.
- 기존 카드 클릭 시 상세 페이지 이동 동작은 변경하지 않는다.
- 스퀴즈 레벨 0(NO SQ)은 라벨로 표시하지 않는다 (정상 상태이므로 강조 불필요).
- 섹터 데이터는 국내 stock_master 테이블에서 조회하며, 미국 종목은 시장 유형(SP500/NQ100/R1000)이 섹터를 대체한다.
- 컬러 테마 변경은 CSS 변수 수정으로 전체 페이지에 일괄 적용된다 (페이지별 개별 수정 불필요).
- 지표 조건 라벨 중요도 순서: 신호 강도(STRONG BUY/SELL) > 스퀴즈 레벨 > RSI 상태 > BB %B 상태 > 거래량 > MACD.

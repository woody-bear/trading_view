# Feature Specification: BUY 사인조회 주식목록 페이지

**Feature Branch**: `009-buy-scan-watchlist`
**Created**: 2026-03-30
**Status**: Draft
**Input**: 상단에 새로운 페이지와 탭을 추가 — BUY 사인조회 주식목록. 표형식으로 국내(코스피/코스닥/ETF) + 미국(나스닥/NYSE ETF) 전 종목 나열. 종목 검색창. 스캔 스케줄 상황판(국내 7회 + 미국 2회). 총 스캔 종목수 표시.

## User Scenarios & Testing *(mandatory)*

### User Story 1 — 전체 스캔 대상 종목 목록 확인 (Priority: P1)

사용자가 "어떤 종목들이 BUY 신호 스캔 대상인지" 한눈에 확인하고 싶다. 시장별·카테고리별로 분류된 전체 종목 리스트를 표 형식으로 보여준다.

**Why this priority**: 사용자가 스캔 범위를 신뢰하려면 실제 대상 종목 목록을 직접 확인할 수 있어야 한다. 가장 핵심 기능.

**Independent Test**: BUY조회 페이지 접근 시 국내/미국 탭이 나타나고 각 카테고리별 종목 수(코스피 934, 코스닥 1784, 코스피ETF 870, 미국주식 107, 미국ETF 84 등)와 종목 리스트가 표시되면 성공.

**Acceptance Scenarios**:

1. **Given** 사용자가 BUY조회 페이지에 접근하면, **When** 페이지가 로드되면, **Then** 국내/미국 탭과 각 카테고리별 종목 수 요약이 상단에 표시된다
2. **Given** 국내 탭이 선택된 상태에서, **When** 탭 콘텐츠를 확인하면, **Then** 코스피(일반주/ETF), 코스닥 종목이 표 형식으로 나열된다
3. **Given** 미국 탭이 선택된 상태에서, **When** 탭 콘텐츠를 확인하면, **Then** NASDAQ 일반주, NYSE ETF 종목이 표 형식으로 나열된다
4. **Given** 종목 테이블에서, **When** 종목명을 클릭하면, **Then** 해당 종목의 상세 페이지로 이동한다

---

### User Story 2 — 종목 검색 (Priority: P2)

3,700개 이상의 종목 중 특정 종목이 스캔 대상인지 빠르게 찾고 싶다.

**Why this priority**: 리스트가 수천 건이므로 스크롤 탐색은 비현실적. 검색창이 없으면 목록의 실용성이 크게 떨어진다.

**Independent Test**: 검색창에 "삼성" 입력 시 종목명 또는 코드에 "삼성"이 포함된 행만 필터링되어 표시되면 성공.

**Acceptance Scenarios**:

1. **Given** 검색창이 비어있으면, **When** 페이지를 로드하면, **Then** 전체 종목이 표시된다
2. **Given** 검색창에 종목명 일부를 입력하면, **When** 입력과 동시에, **Then** 종목명 또는 종목코드에 검색어가 포함된 행만 실시간 필터링된다
3. **Given** 검색 결과가 없는 키워드를 입력하면, **When** 필터링 후, **Then** "검색 결과 없음" 메시지가 표시된다
4. **Given** 검색 중 탭을 전환하면, **When** 탭 전환 후, **Then** 검색어는 유지되고 해당 탭 기준으로 필터링된다

---

### User Story 3 — 스캔 스케줄 상황판 확인 (Priority: P3)

오늘 총 몇 번의 스캔이 언제 실행됐는지, 현재 진행 중인 스캔이 있는지 확인하고 싶다.

**Why this priority**: 언제 스캔이 돌았는지 알아야 BUY 신호 결과의 최신성을 판단할 수 있다.

**Independent Test**: 상황판에 국내 스캔 스케줄(9:30~15:30, 7회)과 미국 스캔 스케줄(19:50, 03:50)이 표시되고, 각 시간대의 완료/진행중/예정 상태가 구별되면 성공.

**Acceptance Scenarios**:

1. **Given** 상황판이 표시되면, **When** 평일 장중에 접근하면, **Then** 국내 스캔 7회(9:30/10:30/11:30/12:30/13:30/14:30/15:30), 미국 스캔 2회(19:50/03:50) 슬롯이 모두 표시된다
2. **Given** 과거에 완료된 스캔 슬롯은, **When** 상황판을 보면, **Then** 완료 상태(완료 시각, 스캔 종목수, BUY 발견 수)가 표시된다
3. **Given** 현재 진행 중인 스캔이 있으면, **When** 상황판을 보면, **Then** 해당 슬롯이 "진행중" 상태로 강조 표시된다
4. **Given** 아직 실행 전인 스캔 슬롯은, **When** 상황판을 보면, **Then** "예정" 상태로 표시된다

---

### User Story 4 — 총 스캔 종목수 요약 (Priority: P3)

스캔 시스템이 총 몇 개의 종목을 커버하는지 한눈에 확인하고 싶다.

**Why this priority**: 시스템 신뢰도 확인용. 간단한 숫자 표시이지만 사용자가 요청한 정보.

**Independent Test**: 페이지 상단에 "총 X개 종목 스캔 중" 문구와 카테고리별 분류가 표시되면 성공.

**Acceptance Scenarios**:

1. **Given** 페이지 접근 시, **When** 페이지가 로드되면, **Then** 국내 총 X개 + 미국 총 Y개 = 전체 Z개 형식으로 표시된다
2. **Given** 국내/미국 카테고리별 요약에서, **When** 각 항목을 보면, **Then** 코스피 일반주, 코스피 ETF, 코스닥, NASDAQ, NYSE ETF 각각의 종목 수가 정확히 표시된다

---

### Edge Cases

- stock_master 데이터 조회 실패 시 "데이터 로딩 중..." 표시 후 재시도
- 검색어에 특수문자(/, ., $) 포함 시 정상 처리
- 3,700개 이상의 종목 목록을 가상 스크롤 또는 페이지네이션으로 성능 저하 없이 표시
- 스캔 이력이 없는 날(주말/공휴일) 접근 시 "오늘 스캔 기록 없음" 표시
- 모바일에서 6탭으로 확장 시 탭 레이블이 잘리지 않도록 처리

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 시스템은 PC 상단 네비게이션과 모바일 하단탭에 "BUY조회" 탭을 추가해야 한다
- **FR-002**: 페이지 최상단에 총 스캔 종목수 요약(국내 X개 + 미국 Y개 = 전체 Z개)을 표시해야 한다
- **FR-003**: 페이지 상단에 스캔 스케줄 상황판을 표시해야 한다 — 국내 7회(9:30/10:30/11:30/12:30/13:30/14:30/15:30 KST), 미국 2회(19:50/03:50 KST)
- **FR-004**: 각 스캔 슬롯은 완료/진행중/예정 세 가지 상태를 색상 또는 아이콘으로 시각적으로 구분해야 한다
- **FR-005**: 완료된 스캔 슬롯에는 완료 시각, 스캔한 종목 수, 발견된 BUY 신호 수를 표시해야 한다
- **FR-006**: 검색창에 입력 시 종목명 또는 종목코드 기준으로 실시간 필터링해야 한다
- **FR-007**: 국내/미국 탭으로 시장을 구분하여 종목 목록을 표시해야 한다
- **FR-008**: 국내 탭은 코스피 일반주, 코스피 ETF, 코스닥 세 카테고리로 분류하여 각 카테고리별 종목 수를 헤더에 표시해야 한다
- **FR-009**: 미국 탭은 NASDAQ 일반주, NYSE ETF 두 카테고리로 분류하여 각 카테고리별 종목 수를 헤더에 표시해야 한다
- **FR-010**: 종목 목록은 표(테이블) 형식으로 종목코드, 종목명, 시장구분을 표시해야 한다
- **FR-011**: 종목 행 클릭 시 해당 종목 상세 페이지로 이동해야 한다
- **FR-012**: 대량 종목(3,700개+) 표시 시 성능 저하 없이 스크롤 가능해야 한다

### Key Entities

- **스캔 슬롯**: 예정 시각(KST), 상태(완료/진행중/예정), 완료 시각, 스캔 종목수, BUY 발견수
- **종목 행**: 종목코드, 종목명, 시장 구분(코스피/코스닥/NASDAQ/NYSE), ETF 여부
- **카테고리 요약**: 카테고리명, 해당 종목 수

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 사용자가 특정 종목이 스캔 대상인지 검색으로 5초 이내 확인 가능
- **SC-002**: 페이지 로드 후 3초 이내 전체 종목 목록이 표시됨
- **SC-003**: 스캔 상황판이 오늘 실행된 모든 스캔 결과를 누락 없이 표시
- **SC-004**: 3,700개 이상의 종목 목록에서 검색 필터링이 타이핑과 동시에 즉각 반응
- **SC-005**: 국내/미국 전 카테고리 종목 수 합계가 실제 데이터와 100% 일치

## Assumptions

- `stock_master` 테이블의 `market_type` + `is_etf` 컬럼 기준으로 분류
- 현재 데이터 기준: 코스피 일반주 934개, 코스피 ETF 870개, 코스닥 1784개, NASDAQ 107개, NYSE ETF 84개 (총 3,779개)
- 스캔 스케줄 상황판은 기존 `/api/scan/full/history` 및 `/api/scan/full/status` API를 재사용
- 종목 목록 데이터는 새 전용 API 엔드포인트(`/api/scan/symbols`)를 통해 stock_master 전체를 반환
- 모바일 하단탭에 "BUY조회" 추가 시 6탭이 됨 — 아이콘 크기 및 텍스트 레이아웃 조정 필요
- 실제 스캔 스케줄: 국내 평일 매시 :30 (9시~15시, 7회) + 미국 19:50, 03:50 (총 9회 슬롯)
- 종목 목록은 브라우저 내 클라이언트 필터링으로 구현 (서버 재검색 불필요)

# Contract: Claude Context Loading

**Consumer**: Claude Code agent (이 저장소에서 실행되는 모든 Claude 에이전트).
**Provider**: `.claude/` 디렉터리 + `CLAUDE.md`의 트리거 표.

## Rule 1 — 트리거 표 우선 조회

Claude는 사용자 요청을 받으면 다음 순서로 동작해야 한다:

1. 요청의 **작업 유형**을 식별.
2. `CLAUDE.md` 섹션 4(트리거 표)에서 해당 유형을 조회.
3. 지정된 `files`만 읽는다(최대 3개).
4. 그 외 `.claude/` 문서는 **읽지 않는다**(명시적 필요가 없는 한).

**Violation**: 트리거 표를 조회하지 않고 `.claude/`를 광범위 스캔하면 계약 위반.

## Rule 2 — 트리거 표 미스 시 처리

작업 유형이 트리거 표에 **없는 경우**:

1. 추측으로 `.claude/` 문서를 열거나 신규 작성하지 않는다.
2. 사용자에게 다음을 보고한다:
   - 요청한 작업 유형.
   - 가장 가까운 기존 표 엔트리(있다면).
   - "트리거 표에 이 유형을 추가해야 합니다"라는 제안.
3. 사용자 결정을 기다린다.

## Rule 3 — 보호된 규칙 편집 시

`path`가 `.claude/rules/`로 시작하는 문서를 Claude가 편집하려 할 때:

1. 편집을 **시작하기 전**에 사용자에게 변경 요약·파급 효과를 보고.
2. 사용자 승인을 명시적으로 받는다(묵시 승인 금지).
3. 승인 기록을 PR 설명(또는 대화 맥락)에 남긴다.

## Rule 4 — 메타 헤더 존중

ContextDocument를 수정할 때:

1. YAML frontmatter의 `last-audit` 필드를 수정 일자로 갱신한다.
2. `update-trigger`가 기술된 조건 중 하나에 해당하는 변경인지 확인.
3. 갱신이 필요 없다고 판단되면 그 근거를 커밋 메시지에 적는다.

## Rule 5 — archive 문서 참조 금지

`.claude/archive/` 하위 문서는 **읽기 전용 역사적 자료**로만 취급:

- 트리거 표가 archive 문서를 가리키면 계약 위반 → 트리거 표 갱신 필요.
- Claude는 스스로 archive 문서를 컨텍스트로 포함시키지 않는다.

## Verification

- **SC-001**에 의해 측정: 임의 5작업에서 평균 `files` 로드 ≤2, 성공률 80%.
- 위반 사례 발견 시 본 계약 또는 트리거 표를 갱신하는 PR을 연다.

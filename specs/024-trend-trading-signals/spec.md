## Background

종목 상세화면의 차트 분석 탭에 **현재 종목의 주가 추세를 분류**하고, 그에 따른 **매수급소·매도시점**을 명시적으로 표시하는 섹션을 추가한다. 판단 근거는 **순수한 가격 추세선(고점·저점 연결)** — RSI·MACD·BB·EMA 같은 보조지표는 사용하지 않는다.

참고 PDF의 4가지 추세 분류 기준을 채택:

### 매수 시점 (4가지 추세)
| 추세 | 매수 급소 |
|------|----------|
| ① 상승추세 | 상승 지지선 지점 또는 상승 저항선을 돌파하는 지점 |
| ② 하락추세 | 지지선 지점 — 단, 추세가 강한 경우 일단 관망 |
| ③ 평행(보합)추세 | 박스권 저점 지지선 또는 박스권 상향 돌파 지점 |
| ④ 삼각수렴(쐐기형) | 고점 추세선을 돌파하는 지점 — 최고의 매수시점 |

### 매도 시점 (4가지 추세)
| 추세 | 매도 시점 |
|------|----------|
| ① 하락추세 | 고점 저항선 지점 또는 저점 지지선이 무너지는 지점 |
| ② 상승추세 | 고점 저항선 지점(1차) 또는 저점 추세선 하향 이탈 지점 |
| ③ 평행(보합)추세 | 박스권 고점대 지점 또는 박스권 하향 이탈 지점 |
| ④ 삼각수렴(쐐기형) | 저점 추세선 하향 이탈 — 강한 매도시점 |

## User Scenarios & Testing *(mandatory)*

### User Story 1 — 추세 자동 분류 + 시각화 (Priority: P1)

차트 분석 탭을 연 사용자가 화면 상단(또는 차트 직하)의 새 섹션에서 **현재 종목이 어떤 추세에 속하는지(상승·하락·평행·삼각수렴)** 한눈에 확인한다. 추세 라벨은 색·아이콘으로 직관적으로 구분된다.

**Why this priority**: 본 기능의 토대. 추세가 무엇인지 정해져야 매매 시점도 의미를 가짐.

**Independent Test**: 상승추세가 명확한 종목, 하락추세 종목, 박스권 종목, 삼각수렴 종목 4종에 대해 라벨이 각각 ①상승 / ②하락 / ③평행 / ④삼각수렴으로 표시되는지 확인.

**Acceptance Scenarios**:
1. **Given** 60일 이상 우상향 종목, **When** 차트 분석 탭 진입, **Then** "📈 상승추세" 라벨 표시.
2. **Given** 60일 이상 우하향 종목, **When** 차트 분석 탭 진입, **Then** "📉 하락추세" 라벨.
3. **Given** 박스권 횡보 종목, **When** 차트 분석 탭 진입, **Then** "↔ 평행(보합)" 라벨.
4. **Given** 고점·저점 수렴 중인 종목, **When** 차트 분석 탭 진입, **Then** "▶ 삼각수렴" 라벨.

---

### User Story 2 — 매수급소 표시 (Priority: P1)

분류된 추세에 맞춰 **현재 가격 기준의 매수 급소가 어디인지** 가격값 + "지금 매수 가능 / 관망" 등의 액션 라벨이 함께 표시된다.

**Why this priority**: 추세 정보의 실질적 활용. "지금 사도 되는지" 직관적 답.

**Independent Test**: 상승추세 종목에서 "상승 지지선의 가격 X원" + "현재가가 지지선 근접 → 매수 후보" 같은 안내가 표시되는지.

**Acceptance Scenarios**:
1. **Given** 상승추세 + 현재가가 지지선 위 안전, **Then** "🟢 매수 가능 — 지지선 X원 근처 매수 적합".
2. **Given** 상승추세 + 저항선 돌파 직후, **Then** "🟢 매수 가능 — 저항선 돌파 매수".
3. **Given** 하락추세 + 추세 강세, **Then** "🟡 관망 — 하락 추세가 강해 추가 하락 가능".
4. **Given** 평행(박스권), **Then** "🟢 매수 가능 — 박스 하단 X원" 또는 "🟢 박스 상단 돌파 시 매수".
5. **Given** 삼각수렴, **Then** "🟢 매수 가능 — 고점 추세선 돌파 시 최고 매수시점".

---

### User Story 3 — 매도시점 표시 (Priority: P1)

분류된 추세에 맞춰 **현재 보유 중이라면 어디가 매도시점인지** 1·2단계로 명시한다.

**Why this priority**: 매수와 함께 매매 사이클 완성.

**Independent Test**: 상승추세 종목에서 "1차 매도가 X원(저항선) / 2차 매도가 Y원(저점 추세선 이탈)" 두 단계 안내 표시.

**Acceptance Scenarios**:
1. **Given** 상승추세 보유자, **Then** "🔴 1차 매도 — 저항선 X원" + "2차 매도 — 저점 추세선 Y원 이탈 시".
2. **Given** 하락추세, **Then** "🔴 매도 — 고점 저항선 X원" + "🔴 강한 매도 — 저점 지지선 Y원 붕괴 시".
3. **Given** 평행(박스권), **Then** "🔴 매도 — 박스 상단 X원" + "🔴 강한 매도 — 박스 하단 Y원 이탈 시".
4. **Given** 삼각수렴, **Then** "🔴 강한 매도 — 저점 추세선 Y원 하향 이탈 시".

---

### User Story 4 — 차트 위 추세선·기준선 시각화 (Priority: P2)

자동 산출된 **상승/하락 추세선·박스 상하단·삼각수렴 라인**이 차트(메인 캔들 차트) 위에 점선·실선으로 오버레이되어, 텍스트 안내와 시각적으로 일치한다.

**Why this priority**: 텍스트 + 시각의 결합으로 직관성을 극대화.

**Independent Test**: 추세선이 차트의 실제 고/저점을 잘 잇고 있는지 시각 검수.

**Acceptance Scenarios**:
1. **Given** 상승추세 분류, **Then** 저점들을 잇는 상승 지지선 + 고점들을 잇는 상승 저항선이 점선으로 차트에 표시.
2. **Given** 평행추세 분류, **Then** 박스 상단·하단 가격이 수평선으로 표시.
3. **Given** 삼각수렴 분류, **Then** 수렴하는 두 추세선이 표시.

---

### Edge Cases

- **데이터 부족** (60거래일 미만): 추세 판정 불가 → "데이터 부족" 안내.
- **명확한 추세 없음**: 어느 패턴에도 부합도 낮으면 "분류 불가 — 명확한 추세 없음" 안내, 매매 시점 미표시.
- **갭(공휴일·정지)**: 거래정지 후 재개 등 가격 점프는 추세 산출에서 보정하거나 별도 표기.
- **암호화폐·ETF**: 추세 분석은 가능하나 해석 위험 — 종목 유형 표기는 그대로 진행.
- **시간 프레임**: 일봉(1d) 기준만 1차 지원. 주봉·월봉은 향후.
- **과거 시점 분석 미지원**: 섹션은 "현재 시점" 분류만 표시.

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 차트 분석 탭에 **신규 "추세 분석" 섹션**을 추가한다. 위치는 **EmaOnlyChart 직하단**(메인 캔들 차트 → EMA 보조 차트 → 추세 분석 카드 → 기존 분할매수 가이드 순). 카드 1개 안에 ① 추세 라벨 → ② 매수 후보 → ③ 매도 후보 수직 배치, 우상단에 "추세선 표시" 토글, 하단에 면책 문구 1줄. 컨테이너는 `bg-[var(--card)] border border-[var(--border)] rounded-lg`로 가치 탭/분할매수 가이드와 동일 패턴, 폰트는 Tailwind 의미 스케일 그대로 따른다.
- **FR-002**: 종목의 **일봉(1d) 최근 120봉(약 6개월)** 가격 흐름만을 사용해 추세를 4종 중 하나로 자동 분류: ① 상승추세, ② 하락추세, ③ 평행(보합), ④ 삼각수렴. 분류 불가 시 "분류 불가" 상태.
- **FR-003**: 추세 분류 결과는 **색·아이콘·라벨 텍스트**로 시각화한다.
  - 상승추세 = `text-[var(--buy)]` 초록 + 📈
  - 하락추세 = `text-[var(--sell)]` 빨강 + 📉
  - 평행(보합) = `text-[var(--neutral)]` 파랑/회색 + ↔
  - 삼각수렴 = `text-yellow-400` 노랑 + ▶
  - 라벨 타이포: `text-sm font-semibold`.
- **FR-004**: 분류된 추세에 대응하는 **매수 후보 구간**(가격 또는 조건)과 **상태 라벨**이 표시된다. 톤은 중립적(📍 매수 후보 구간 / 🟡 관망 등) — 투자 권유형 표현 회피. PDF의 매수 시점 기준을 따른다.
- **FR-005**: 분류된 추세에 대응하는 **매도 후보 구간 1·2단계**(가격 또는 조건)가 표시된다. 중립 톤(⚠️ 매도 후보 / ⚠️ 강한 매도 후보). PDF의 매도 시점 기준을 따른다.
- **FR-006**: 추세 판정에 사용된 **상승/하락 추세선, 박스 상·하단, 삼각수렴 라인**은 새 섹션의 **"추세선 표시" 토글**(기본 OFF)로 ON 시 메인 캔들 차트 위에 점선/실선으로 오버레이된다. 기본은 OFF로 차트 가독성을 유지한다.
- **FR-007**: 데이터가 120거래일 미만이면 추세 분석을 건너뛰고 "데이터 부족" 안내만 표시.
- **FR-008**: 명확한 추세에 부합하지 않으면 "분류 불가" 상태로 표시하고 매매 시점 미노출.
- **FR-009**: 추세·매수·매도 가격 정보는 **최신 일봉 마감 시점 기준**으로 산출되며, 종목 진입·새로고침 시 갱신. 실시간 가격 변동마다 재계산하지 않는다.
- **FR-010**: 본 섹션은 **차트 분석 탭에만** 노출. 가치 분석 탭 등 무영향.
- **FR-011**: 산출된 매매 시점이 현재가와 가까울 경우 (±2% 이내) **시각적 강조**(아이콘·배경 색)로 주의 환기.
- **FR-012**: 매매 안내는 참고용이라는 **면책 문구**가 섹션에 1회 표시된다.
- **FR-013**: 본 기능은 **종목 상세 차트 분석 탭의 수동 사용자 진입에만 동작**한다. 기존 시스템(전체 스캔·BUY 신호 판정·차트 BUY/SELL 마커·스냅샷·스케줄러·BUY 알림·`.claude/rules/*.md`)에 **어떠한 변경·참조도 없어야** 하며, 본 기능 도입으로 BUY 신호 결과가 1건이라도 달라지면 안 된다(회귀 0건).

### Key Entities

- **TrendClassification**: 종목별 현재 추세 분류 결과. 속성: 종목코드, 시장, 분류일, 추세타입(uptrend/downtrend/sideways/triangle/unknown), 분류 신뢰도, 분석 윈도우 봉수.
- **TrendLine**: 차트에 그려질 추세선. 속성: 라인 종류(상승지지선·상승저항선·하락지지선·하락저항선·박스상단·박스하단·삼각수렴선), 시작·끝 시점·가격, 표시 스타일.
- **TradingSignal**: 추세 기반 매매 시점. 속성: 종류(매수가능/관망/매도1차/매도2차/강한매도), 기준 가격, 조건 텍스트, 현재가 대비 거리(%).

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 차트 분석 탭 진입 후 **3초 이내** 추세 라벨·매매 시점 안내 확인 가능.
- **SC-002**: KR·US 개별주식 샘플 30종목 분류가 **육안 검수와 80% 이상 일치**.
- **SC-003**: 매수급소·매도시점 가격이 차트 추세선과 **시각적으로 일치** (오버레이 ±1% 이내).
- **SC-004**: 데이터 부족·분류 불가 종목에서 **혼동 없는 안내 문구**, 매매 시점 잘못 노출 0건.
- **SC-005**: 본 섹션 추가로 인한 차트 분석 탭 **최초 렌더 지연 0.3초 이하**.
- **SC-006**: 사용자 매매 시점 안내 만족도 **70% 이상** (도움됨 이상).

## Assumptions

- 분석 데이터는 기존 chart 응답의 일봉(1d) OHLC를 그대로 사용 — 별도 데이터 소스 없음.
- 추세선 산출은 **고점·저점 피크 검출 + 선형 회귀** 기반. 1차 구현은 단순 룰 기반(예: 최근 N봉의 고점 2개·저점 2개 연결) 후 정교화.
- "삼각수렴"은 상승 추세선과 하락 추세선이 교차 직전인 패턴 — 저점 상승 + 고점 하강.
- "박스권"은 일정 가격대 내 횡보 — 최근 N봉 변동폭이 좁은 경우.
- 임계값(피크 윈도우·기울기 임계 등)은 디폴트로 시작, SC-002 기준 튜닝.
- 매매 시점은 **참고용**, 투자 책임은 사용자.
- 본 기능은 일봉만 지원 (주봉·월봉 미지원).

## Out of Scope (명시적 비포함)

- 분/시간봉 기반 추세 분석
- 다중 시간 프레임 일치 분석
- 머신러닝 기반 추세 예측
- 백테스트 결과 표시
- 자동 매매 연동
- 알림(텔레그램·이메일) 발송
- 가치 분석 탭 등 다른 탭으로의 노출

## Dependencies

- 022 종목 상세 차트 분석 탭 안정 운영.

# Feature Specification: 종목 상세화면 2단 분석 뷰 (1차 차트 → 2차 가치)

**Feature Branch**: `022-stock-detail-layered-analysis`
**Created**: 2026-04-14
**Status**: Draft
**Input**: User description: "종목상세화면을 1차 차트분석 > 2차 가치분석으로 볼수있도록 개선"

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 차트 분석을 1차 화면으로 즉시 확인 (Priority: P1)

사용자가 관심종목·스캔결과·BUY 사례 등에서 특정 종목을 선택하면, 현재와 동일한 차트 중심의 기술적 분석 화면(캔들, BB, RSI, MACD, 스퀴즈, BUY/SELL 마커 등)이 **기본(1차) 뷰**로 노출된다.

**Why this priority**: 기존 사용자 플로우(신호 확인 → 매매 판단)의 핵심 경로이므로 회귀가 없어야 한다. 2차 뷰가 추가되어도 1차 뷰 진입에는 추가 클릭/탭이 없어야 한다.

**Independent Test**: 기존에 동작하던 모든 차트 분석 기능이 동일 위치·동일 동작으로 유지되는지 확인. 상세 진입 시 첫 화면이 차트 분석이며 추가 클릭 없이 접근 가능함을 검증.

**Acceptance Scenarios**:

1. **Given** 사용자가 대시보드/스캔/스크랩 목록에서 종목을 클릭, **When** 상세화면이 열림, **Then** 1차 탭(차트 분석)이 기본 선택 상태로 표시된다.
2. **Given** 1차 탭이 활성화됨, **When** 사용자가 기간·민감도·지표 토글을 조작, **Then** 기존과 동일하게 차트가 갱신된다.
3. **Given** URL로 직접 상세 진입, **When** 별도 쿼리파라미터가 없음, **Then** 1차(차트) 탭이 기본 노출된다.

---

### User Story 2 - 가치분석을 2차 화면으로 전환해 확인 (Priority: P1)

사용자가 상세화면에서 "가치 분석" 탭을 눌러 전환하면, 해당 종목의 재무·밸류에이션 지표를 한 화면에서 비교·검토할 수 있다.

**Why this priority**: 매수 판단을 "기술적 신호 + 펀더멘털" 2단계로 의사결정하게 하는 핵심 가치 제안. 탭만 새로 추가되고 데이터만 적절히 노출되면 독립적으로 MVP 가치가 성립.

**Independent Test**: 1차 탭이 존재한다는 가정하에, 2차 탭 클릭만으로 가치 지표 섹션이 표시되고 로딩·빈값·오류 상태가 구분되는지 검증.

**Acceptance Scenarios**:

1. **Given** 상세화면이 열려 있음, **When** 사용자가 "가치 분석" 탭을 클릭, **Then** 종목의 가치지표 요약이 표시된다.
2. **Given** 가치 분석 탭 활성화, **When** 데이터가 로드 중, **Then** 스켈레톤·로딩 인디케이터가 노출된다.
3. **Given** 가치 분석 탭 활성화, **When** 해당 종목의 가치 데이터를 수집할 수 없음(예: ETF·암호화폐), **Then** "해당 종목은 가치 분석 대상이 아닙니다" 안내가 표시된다.
4. **Given** 가치 분석 탭에 진입한 상태, **When** 사용자가 1차 탭으로 되돌아감, **Then** 차트 상태(기간·지표 토글)가 유지된 채 복원된다.

---

### User Story 3 - 탭 간 전환 시 URL·뒤로가기 일관성 (Priority: P2)

사용자가 탭을 전환할 때 URL(또는 쿼리)이 갱신되고, 브라우저 뒤로가기로 이전 탭으로 되돌아갈 수 있다. 공유 링크로 2차 탭을 직접 열 수도 있다.

**Why this priority**: 네비게이션 예측 가능성·공유성 확보. 핵심 기능은 아니지만 품질·UX 완성도 항목.

**Independent Test**: 탭 전환 후 URL이 바뀌는지, 해당 URL을 새 탭에서 열었을 때 그 탭이 활성화된 상태로 진입하는지 확인.

**Acceptance Scenarios**:

1. **Given** 1차 탭 활성화 상태, **When** 가치 분석 탭으로 전환, **Then** URL 쿼리(예: `?tab=value`)가 반영된다.
2. **Given** `?tab=value`가 포함된 URL로 진입, **When** 화면 로드 완료, **Then** 가치 분석 탭이 기본 선택된 상태로 열린다.
3. **Given** 탭 전환 후, **When** 브라우저 뒤로가기 실행, **Then** 이전 탭으로 되돌아간다.

---

### Edge Cases

- **가치 데이터 미지원 종목**: ETF, 암호화폐(BTC/ETH 등), 지수 등은 재무 데이터가 없으므로 "가치 분석 미지원" 안내를 명시해야 한다.
- **일부 지표만 수집됨**: PER은 있는데 PBR이 없는 경우 등 — 결측 지표는 "-"로 표시하고 다른 지표는 정상 노출한다.
- **외화 표기**: US 종목은 USD 기준, KR 종목은 KRW 기준으로 자연스럽게 표기한다.
- **오래된 스냅샷**: 공시·재무 데이터가 최근 1분기·연간 기준임을 기준일과 함께 표시한다.
- **모바일 레이아웃**: 탭이 상단에 고정되고 본문만 스크롤되도록 유지되어야 한다 (BottomNav와 중첩 금지).
- **빠른 탭 반복 클릭**: 탭 전환 중 중복 요청이 누적되지 않도록 마지막 요청만 반영되어야 한다.

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 상세화면은 **2개의 탭**(①차트 분석, ②가치 분석)을 제공하고, 진입 시 차트 분석 탭이 기본으로 활성화되어야 한다.
- **FR-002**: 차트 분석 탭은 기존 기술적 분석 기능(캔들/BB/RSI/MACD/스퀴즈/신호 마커/기간·민감도 토글)을 동일 동작·레이아웃으로 유지해야 한다 (회귀 금지).
- **FR-003**: 가치 분석 탭은 다음 지표를 카드형 요약으로 **중요도 순**으로 제공해야 한다: ① 시가총액 → ② PER → ③ PBR → ④ ROE → ⑤ EPS → ⑥ 배당수익률 → ⑦ 섹터(업종).
- **FR-004**: 가치 분석 탭은 업종/시장 평균 벤치마크가 **수집되어 있는 경우에 한해** 각 지표 옆에 해석 라벨(저평가·평균·고평가)을 표시한다. **본 1차 범위에서는 벤치마크 데이터를 수집하지 않으므로** 모든 카드는 수치만 표시하고 라벨은 노출하지 않는다. (벤치마크 도입은 별도 후속 기능)
- **FR-005**: 탭 전환 시 사용자 URL 상태가 반영되고, 뒤로가기·앞으로가기로 이전 탭 상태로 이동할 수 있어야 한다.
- **FR-006**: 가치 분석 탭은 **KR·US 개별 상장 주식**만 지원한다. 그 외 자산군(ETF·암호화폐·지수·외환 등)에서는 탭을 **비활성(disabled, 회색)** 상태로 노출하고, hover 시 툴팁(PC) 또는 탭 시 토스트(모바일)로 **"ETF·암호화폐·지수·외환은 가치 분석 미지원입니다"** 안내를 표시해야 한다.
- **FR-007**: 가치 데이터 카드 영역에는 두 가지 시점을 분리하여 표시해야 한다.
  - **보고 기준일(reporting_period)**: 최신 분기 또는 회계연도 종료일 (예: "2025-Q4 기준" 또는 "2024-12-31"). 데이터 소스가 제공하는 `mostRecentQuarter` / `lastFiscalYearEnd` 등에서 추출.
  - **최종 갱신 시각(cached_at)**: 서버가 외부 소스에서 응답을 캐시한 시각 (UTC ISO-8601). 헤더 또는 푸터의 보조 영역에 작게 표기.
  - 둘 중 보고 기준일이 결측이면 해당 라벨만 생략하고 갱신 시각은 항상 표기.
- **FR-008**: 가치 데이터는 종목별로 캐싱되어 탭 재전환 시 즉시 복원되며, 일정 주기마다 재검증되어야 한다.
- **FR-009**: 모바일 레이아웃에서 탭 스트립은 상단 고정되고, 본문만 스크롤되어야 한다. BottomNav 영역과 겹치지 않아야 한다.
- **FR-012**: 모바일 가치 분석 탭의 본문은 **CSS Scroll Snap(`scroll-snap-type: y mandatory`)**으로 동작하며, 각 지표 섹션(또는 섹션 묶음)의 시작 지점에 스냅(`scroll-snap-align: start`)되어야 한다. PC에서는 일반 스크롤로 동작한다.
- **FR-010**: 1차 ↔ 2차 탭 전환 시 각 탭이 가진 사용자 조작 상태(차트 기간·지표 토글 등)는 세션 동안 유지되어야 한다.
- **FR-011**: 가치 분석 탭 내 각 지표 카드는 툴팁 또는 간단 설명을 제공해 비전문 사용자도 의미를 이해할 수 있어야 한다.

### Key Entities *(include if feature involves data)*

- **ValuationSnapshot**: 특정 종목의 특정 시점 가치 지표 스냅샷. 속성: 종목코드, 시장, 기준일(분기·연간), 시가총액, PER, PBR, EPS, ROE, 배당수익률, 섹터, 갱신시각, 데이터 출처.
- **SectorBenchmark** *(선택)*: 업종·시장별 각 지표의 중앙값 또는 평균. 사용자 종목의 상대 해석(저평가·고평가)을 위한 참고치.
- **DetailView 상태**: 탭 키(`chart` | `value`) + 각 탭의 사용자 조작 상태(차트 기간·지표 토글 등). 세션 단위로 복원 대상.

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 상세화면 진입 후 1차(차트) 탭이 **즉시**(0.5초 이내) 표시되며, 기존 차트 분석 기능의 회귀 버그는 0건이다.
- **SC-002**: 가치 분석 탭으로 전환 시 지원 종목의 데이터가 **1.5초 이내**(p95)로 노출된다.
- **SC-003**: 가치 분석 탭이 지원되는 종목(KR·US 개별 주식) 중 **90% 이상**에서 FR-003의 **6개 정량 지표(시가총액·PER·PBR·ROE·EPS·배당수익률 — 섹터 제외)** 중 **5개 이상**이 결측 없이 채워진다.
- **SC-004**: 사용자가 매수 판단 플로우(차트 신호 확인 → 가치 확인)를 **3번 이하의 탭·클릭**으로 완료할 수 있다.
- **SC-005**: 공유된 상세 URL을 받은 사용자가 의도한 탭(차트·가치)으로 **추가 조작 없이** 즉시 진입할 수 있다.
- **SC-006**: 가치 분석 미지원 자산군 접근 시 혼동 없이 **명시적 안내 문구**를 받는 사용자 비율 100%.

## Assumptions

- 현재 상세화면(`SignalDetail`)의 기존 차트·지표 UX는 "1차 차트 분석" 뷰의 시작점이며, 기존 컴포넌트 구조를 재사용한다고 가정한다.
- 가치 지표 데이터는 시장별로 상이한 소스(국내·해외)에서 수집 가능하며, 수집 실패·부분 결측은 정상 시나리오로 처리한다.
- 사용자가 주로 판단하는 대표 지표는 PER/PBR/EPS/ROE/시가총액/배당수익률 6종이며, 이를 기본 세트로 시작한다.
- 탭 네비게이션은 URL 쿼리 기반(`?tab=chart|value`)으로 구현해 공유·딥링크를 지원한다.
- 업종 벤치마크는 1차 범위에 포함하되, 벤치마크 데이터가 없을 경우 해석 라벨 없이 수치만 표시해도 수용 가능하다.
- 차트 분석 탭의 기존 동작은 변경 없이 래핑(wrap)되는 것으로 간주하며, 기존 기능 수정·개선은 본 스펙 범위 밖이다.

## Out of Scope (명시적 비포함)

- 재무제표 상세(손익계산서·현금흐름표 전체 표시)
- 경쟁사 다중 종목 비교 보드
- 애널리스트 목표주가·컨센서스
- 뉴스·공시 피드 통합
- 사용자가 직접 입력하는 투자 메모(추후 별도 기능)

## Clarifications

### Session 2026-04-14

- **Q1: 가치 분석 탭의 지원 대상 시장 범위** → **KR + US 개별 주식** (S&P500·NASDAQ100 포함).
  - 지원: KR 상장 주식(코스피200·코스닥150 등), US 상장 개별주식(S&P500·NASDAQ100 범위).
  - 미지원: ETF, 암호화폐, 지수, 외환 — 탭 진입 시 "가치 분석 미지원" 안내 표시 (FR-006).
  - 데이터 소스: 국내·해외 2개 소스 필요 (KR·US 각각 별도 수집 경로).
- **Q2: 미지원 자산군에서의 "가치 분석 탭" 표시 방식** → **disabled(회색) 탭 + 툴팁 안내**.
  - "가치 분석" 탭은 항상 자리는 노출하되, 미지원 자산군(ETF·암호화폐·지수·외환)에서는 **비활성(회색) 상태**로 표시한다.
  - 클릭/탭 시도 또는 hover 시 툴팁으로 **"ETF·암호화폐·지수·외환은 가치 분석 미지원입니다"** 안내한다.
  - 모바일에서는 hover가 없으므로, 비활성 탭을 탭하면 동일 문구의 **토스트/스낵바**를 1회 표시한다.
- **Q3: 가치 분석 지표 표시 순서** → **중요도 순**.
  - 순서: ① 시가총액 → ② PER → ③ PBR → ④ ROE → ⑤ EPS → ⑥ 배당수익률 → ⑦ 섹터(업종).
  - 기준일·데이터 출처는 카드 영역과 별개로 헤더 또는 푸터에 표시.
- **Q4: 모바일 가치 분석 탭의 세로 스크롤 동작** → **CSS Scroll Snap (섹션 단위 스냅)**.
  - 가치 분석 본문 컨테이너에 `scroll-snap-type: y mandatory`, 각 섹션 카드(또는 카드 그룹)에 `scroll-snap-align: start` 적용.
  - 사용자가 위/아래로 스와이프하면 다음/이전 섹션 묶음의 시작 지점으로 자석처럼 정렬된다.
  - 상단 탭 스트립과 하단 BottomNav는 고정, 본문만 스냅 스크롤한다 (FR-009 유지).
  - PC에서는 일반 스크롤로 동작 (스냅은 모바일 한정).

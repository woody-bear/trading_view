# 작업계획: 신호 시스템 점검 및 UI 개선

## 작업 1: BUY "지금 사야할 종목" 확인
**상태**: 정상 동작 확인 완료
**결론**: 현재 모든 종목이 NEUTRAL — BUY 신호가 없는 것이 정상

**BUY 발생 조건 (4가지 동시 충족 필수)**:
| 조건 | 기준 |
|------|------|
| BB %B | ≤ 0.05 (하단밴드 아래) |
| RSI | < 30 (과매도) |
| MACD | 히스토그램 전봉 대비 증가 |
| 거래량 | 1.2배 이상 |

현재 워치리스트 종목들의 RSI가 35~60 범위이므로 BUY 조건 미충족.
BUY 조건이 매우 엄격하여 실제 발생 빈도가 낮음.

**개선 방안**: 조건 완화 옵션 추가 (예: RSI < 40, %B ≤ 0.2) → 사용자 확인 후 진행

## 작업 3: 종합신호 "대기"만 나오는 문제 점검
**파일**: `backend/indicators/signal_engine.py`
**현재 상태**: 모든 종목 NEUTRAL (대기) — 로직은 정상 동작

**신호 상태 3가지**:
- `BUY` (매수) — confidence ≥ 60, 4가지 조건 동시 충족
- `SELL` (매도) — confidence ≥ 60, 4가지 매도 조건 동시 충족
- `NEUTRAL` (대기) — 조건 미충족 또는 복귀

**문제 원인**: BUY/SELL 조건이 매우 엄격 (4가지 동시 충족)
**개선**: 종합신호 영역에 현재 충족된 조건 수를 표시하여 신호 근접 여부를 파악 가능하게 함

**구현**:
- 상세화면에 "신호 조건 충족 현황" 표시 (예: 2/4 조건 충족)
- 각 조건별 충족/미충족 상태를 체크리스트로 표시
- 백엔드: signal_engine에서 개별 조건 충족 여부를 응답에 포함

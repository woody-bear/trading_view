# Quickstart: PC 스캔 화면 BUY 스트립 검증

**Feature**: 032-scan-buy-strip-pc  
**Date**: 2026-04-23

## 사전 조건

- 백엔드 서버 실행 중 (`localhost:8000`)
- 프론트엔드 빌드 완료 (`pnpm build`)
- 최신 스캔 스냅샷 존재 (BUY 신호 종목 포함)

---

## 시나리오 1: BUY 카드 표시 확인 (핵심 경로)

1. PC 브라우저(≥ 768px)에서 스캔 페이지 접속
2. 탭 목록(추천종목·눌림목·대형주) **위**에 BUY 신호 섹션 확인
3. 카드가 최대 5개 가로 그리드(1행)로 표시되는지 확인
4. 각 카드에 종목명·가격·등락률·신호칩·지표칩·Trend·N일전 포함 확인

**기대 결과**: BUY 신호 카드 최대 5개가 탭 위 독립 섹션에 MiniWatchCard 형식으로 표시

---

## 시나리오 2: 탭 전환 시 섹션 유지

1. BUY 섹션이 표시된 상태에서 "눌림목" 탭 클릭
2. BUY 섹션이 사라지지 않고 최상단에 유지되는지 확인
3. "대형주" → "추천종목" 순으로 탭 전환 반복

**기대 결과**: 탭 전환에 관계없이 BUY 섹션 항상 유지

---

## 시나리오 3: 카드 클릭 이동

1. BUY 섹션의 카드 클릭
2. 해당 종목 상세 화면으로 이동하는지 확인

**기대 결과**: 종목 상세 화면으로 정상 이동

---

## 시나리오 4: BUY 신호 없을 때 빈 영역 없음

1. 스냅샷에 BUY 신호가 없거나 API 응답 전 상태 확인
2. 탭 목록이 최상단에 위치하는지 확인 (빈 공간 없음)

**기대 결과**: BUY 섹션 미표시, 빈 공간 없이 탭이 최상단

---

## 시나리오 5: 모바일 미표시 확인

1. 브라우저 창 너비를 768px 미만으로 줄임 (또는 모바일 에뮬레이터)
2. 스캔 페이지에서 BUY 섹션이 표시되지 않는지 확인
3. 모바일 홈 화면의 기존 QuickBuyStrip이 정상 동작하는지 확인

**기대 결과**: 모바일에서 BUY 섹션 미표시, 기존 QuickBuyStrip 동작 유지

---

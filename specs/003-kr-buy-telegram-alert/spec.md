# Feature Specification: 국내주식 BUY 신호 텔레그램 정기 알림

**Feature Branch**: `003-kr-buy-telegram-alert`
**Created**: 2026-03-21
**Status**: Draft
**Input**: User description: "국내주식 당일부터 3일전까지 BUY 사인 발생한 국내 종목 일일2회 ( 오전10시 30분 , 오후 3시) 텔레그램으로 알림보내는 기능"

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 국내주식 BUY 신호 정기 텔레그램 알림 (Priority: P1)

사용자는 매일 오전 10:30과 오후 3:00에 텔레그램으로 국내주식(코스피/코스닥) 중 당일~3일 이내에 BUY 신호가 발생한 종목 목록을 자동으로 받는다. 매매 타이밍을 놓치지 않기 위해 장중에 2회 알림을 받으며, 종목명/가격/신호 강도를 한눈에 확인할 수 있다.

**Why this priority**: 핵심 기능 그 자체. 이 알림 없이는 피처의 가치가 없다.

**Independent Test**: 텔레그램 봇 토큰과 채팅 ID가 설정된 상태에서, 수동으로 알림 전송을 트리거하여 BUY 신호 종목 메시지가 텔레그램에 도착하는지 확인

**Acceptance Scenarios**:

1. **Given** 텔레그램 봇이 설정되어 있고 BUY 신호 종목이 존재할 때, **When** 오전 10:30 또는 오후 3:00이 되면, **Then** 당일~3일 이내 BUY 신호 종목 목록이 텔레그램 메시지로 전송됨
2. **Given** 텔레그램 봇이 설정되어 있고 BUY 신호 종목이 없을 때, **When** 예정 시간이 되면, **Then** "현재 BUY 신호 종목이 없습니다"라는 안내 메시지가 전송됨
3. **Given** 텔레그램 봇이 설정되지 않았을 때, **When** 예정 시간이 되면, **Then** 알림 전송을 건너뛰고 로그에 경고를 남김

---

### User Story 2 - 알림 메시지 포맷 및 정보 제공 (Priority: P1)

사용자가 텔레그램 메시지를 받았을 때, 종목명/심볼/현재가/등락률/신호 강도/신호 발생일을 즉시 파악할 수 있어야 한다. 메시지는 모바일 텔레그램에서도 가독성이 좋아야 하며, 종목을 탭하면 상세화면으로 이동할 수 있는 링크를 포함한다.

**Why this priority**: 알림이 전송되더라도 내용이 불충분하면 사용자가 매매 판단을 내릴 수 없다.

**Independent Test**: 샘플 BUY 신호 데이터로 메시지를 생성하여 텔레그램에 전송한 뒤, 모바일에서 가독성과 링크 동작을 확인

**Acceptance Scenarios**:

1. **Given** BUY 신호 종목 3개가 존재할 때, **When** 알림이 전송되면, **Then** 각 종목별로 종목명, 심볼, 현재가, 등락률(%), 신호 강도(STRONG/NORMAL/WEAK), 신호 발생일이 포함된 메시지가 전송됨
2. **Given** 알림 메시지가 전송되었을 때, **When** 사용자가 종목명을 탭하면, **Then** 해당 종목의 상세화면 URL로 이동함
3. **Given** BUY 신호 종목이 10개 이상일 때, **When** 알림이 전송되면, **Then** 신호 강도 순(STRONG > NORMAL > WEAK)으로 정렬되어 표시됨

---

### User Story 3 - 수동 알림 전송 (Priority: P2)

사용자는 설정 화면에서 "BUY 신호 알림 테스트" 버튼을 눌러 즉시 현재 BUY 신호 목록을 텔레그램으로 전송할 수 있다. 예정 시간을 기다리지 않고 확인하고 싶을 때 사용한다.

**Why this priority**: 정기 알림의 동작 확인 및 디버깅에 필요하지만, 핵심 자동 알림이 우선이다.

**Independent Test**: 설정 화면에서 테스트 버튼을 클릭하여 텔레그램에 메시지가 도착하는지 확인

**Acceptance Scenarios**:

1. **Given** 설정 화면에서 텔레그램이 설정된 상태, **When** "BUY 신호 알림 테스트" 버튼을 클릭, **Then** 현재 BUY 신호 종목 목록이 텔레그램으로 즉시 전송되고 성공 토스트 표시
2. **Given** 텔레그램이 미설정 상태, **When** 테스트 버튼을 클릭, **Then** "텔레그램 설정을 먼저 완료해주세요" 안내 토스트 표시

---

### User Story 4 - 알림 발송 이력 조회 페이지 (Priority: P2)

사용자는 별도 "알림 이력" 페이지에서 과거 BUY 신호 텔레그램 발송 내역을 확인할 수 있다. 발송 시간, 성공/실패 여부, 전송된 종목 수, 실제 전송된 메시지 내용까지 조회하여 알림 기능이 정상 동작하고 있는지 모니터링한다.

**Why this priority**: 핵심 알림 기능(P1) 이후, 운영 모니터링 수단이 필요. 알림이 실패하거나 누락되는 상황을 사용자가 직접 인지해야 한다.

**Independent Test**: 알림이 1회 이상 발송된 후 알림 이력 페이지에 접근하여 발송 기록과 메시지 내용이 표시되는지 확인

**Acceptance Scenarios**:

1. **Given** 알림이 발송된 이력이 있을 때, **When** 알림 이력 페이지에 진입, **Then** 최근 발송 이력이 시간 역순으로 표시됨 (발송 시간, 성공/실패, 전송 종목 수)
2. **Given** 특정 알림 이력을 선택했을 때, **When** 상세 내용을 펼치면, **Then** 실제 텔레그램으로 전송된 메시지 내용(종목 목록)이 표시됨
3. **Given** 알림 전송이 실패한 이력이 있을 때, **When** 이력 목록에서 확인, **Then** 실패 이유(API 오류, 봇 미설정 등)가 표시됨
4. **Given** 알림 이력이 없을 때, **When** 페이지에 진입, **Then** "아직 발송된 알림이 없습니다" 안내 표시

---

### Edge Cases

- 서버 시작 시간이 예정 알림 시간(10:30 또는 15:00)을 이미 지난 경우 → 해당 회차는 건너뛰고 다음 예정 시간에 정상 발송
- 알림 전송 중 텔레그램 API 오류 발생 → 최대 3회 재시도(10초 간격), 실패 시 로그에 에러 기록
- 주말/공휴일에는 장이 열리지 않으므로 → 영업일에만 알림 전송 (주말 건너뜀)
- BUY 신호 종목이 매우 많을 때(30개 이상) → 상위 20개만 전송하고 "외 N개 종목" 표시
- 동일 종목이 여러 날에 걸쳐 BUY 신호 유지 → 가장 최근 신호 발생일 기준 1건만 표시 (중복 제거)

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 시스템은 매일 오전 10:30 KST와 오후 3:00 KST에 국내주식 BUY 신호 알림을 텔레그램으로 자동 전송해야 한다
- **FR-002**: 알림 대상은 당일~3일 이내(자연일)에 BUY 또는 SQZ BUY 신호가 발생한 코스피/코스닥 종목이어야 한다
- **FR-003**: 알림 메시지에는 종목명, 심볼, 현재가, 등락률, 신호 강도(STRONG/NORMAL/WEAK), 신호 발생일이 포함되어야 한다
- **FR-004**: 알림 메시지의 종목명에는 해당 종목 상세화면으로 이동하는 링크가 포함되어야 한다
- **FR-005**: BUY 신호 종목은 신호 강도 순(높은 순)으로 정렬되어야 한다
- **FR-006**: 동일 종목이 여러 날 BUY 신호를 유지한 경우 가장 최근 신호 기준 1건만 표시해야 한다
- **FR-007**: BUY 신호 종목이 없을 때도 "신호 없음" 안내 메시지를 전송해야 한다
- **FR-008**: 텔레그램 봇 미설정 시 알림을 건너뛰고 로그에 경고를 남겨야 한다
- **FR-009**: 주말(토/일)에는 알림을 전송하지 않아야 한다
- **FR-010**: 설정 화면에서 수동으로 BUY 신호 알림을 즉시 전송할 수 있는 버튼을 제공해야 한다
- **FR-011**: 알림 전송 실패 시 최대 3회 재시도(10초 간격)해야 한다
- **FR-012**: 종목 수가 20개를 초과하면 상위 20개만 포함하고 나머지는 "외 N개" 로 요약해야 한다
- **FR-013**: 별도 "알림 이력" 페이지에서 발송 내역(시간, 성공/실패, 종목 수)을 시간 역순으로 조회할 수 있어야 한다
- **FR-014**: 알림 이력에서 개별 항목을 선택하면 실제 전송된 메시지 내용(종목 목록)을 확인할 수 있어야 한다
- **FR-015**: 알림 전송 실패 시 실패 사유(API 오류, 봇 미설정 등)를 이력에 기록해야 한다

### Key Entities

- **BuySignalAlert**: 정기 알림 대상 종목 집합 — 심볼, 종목명, 현재가, 등락률, 신호 상태(BUY/SQZ BUY), 신호 강도, 신호 발생일
- **AlertSchedule**: 알림 발송 일정 — 발송 시간(10:30/15:00 KST), 영업일 여부, 마지막 발송 시각
- **AlertHistory**: 발송 이력 — 발송 시각, 성공/실패, 전송 종목 수, 전송 메시지 내용, 실패 사유

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 영업일 기준 일일 2회(10:30, 15:00) 알림이 1분 이내 오차로 정시에 전송되어야 한다
- **SC-002**: BUY 신호 발생 후 최대 3일(자연일) 이내에 알림에 포함되어야 한다
- **SC-003**: 알림 메시지를 받은 사용자가 5초 이내에 종목 상세정보를 확인할 수 있어야 한다 (링크 탭)
- **SC-004**: 텔레그램 API 정상 시 알림 전송 성공률이 99% 이상이어야 한다
- **SC-005**: 수동 테스트 버튼 클릭 후 10초 이내에 텔레그램 메시지가 도착해야 한다

## Clarifications

### Session 2026-03-21

- Q: 알림 기능이 정상 동작하는지 서비스 화면에서 어떻게 확인하나? → A: 별도 "알림 이력" 페이지에서 발송 내역(시간, 성공/실패, 종목 수) + 전송된 메시지 내용까지 조회 (US4, FR-013~015 추가)

## Assumptions

- 텔레그램 봇 토큰과 채팅 ID는 이미 설정 화면에서 관리되고 있다 (기존 기능)
- BUY/SQZ BUY 신호 데이터는 기존 signal_history 테이블 또는 current_signal 테이블에서 조회 가능하다
- 알림 발송 시간은 한국 표준시(KST, UTC+9) 기준이다
- 서버가 항상 실행 중이며, APScheduler가 크론 작업을 관리한다
- "3일 이내"는 캘린더 기준 3일(영업일 아닌 자연일)이다
- 상세화면 URL 형식은 기존 프론트엔드 라우팅(/{symbol})을 따른다

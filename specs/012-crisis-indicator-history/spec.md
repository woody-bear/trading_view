# Feature Specification: 위기 이벤트 시장 지표 히스토리

**Feature Branch**: `012-crisis-indicator-history`
**Created**: 2026-04-02
**Status**: Draft
**Input**: 전쟁, 펜대믹, 금융위기같은 큰 이슈가 발생했을때는 가장 최근에 과거에 어떻게 주요 지표들이 움직였는지 확인하는 기능

## Overview

전쟁·팬데믹·금융위기 등 글로벌 위기 이벤트 발생 시, 과거 유사 사건에서 주요 시장 지표(주가지수, 채권, 원자재, 환율 등)가 어떻게 움직였는지 비교 조회할 수 있는 기능. 사용자는 현재 상황을 과거 위기 패턴과 대조하여 시장 흐름을 직관적으로 파악하고 투자 판단에 활용한다.

## Clarifications

### Session 2026-04-02

- Q: 비교 차트에서 "현재 진행 중인 이벤트"를 과거 이벤트와 비교하는 방식은? → A: 시스템이 큐레이션 목록에 "진행중" 배지 이벤트를 포함하고, 목록에 없는 경우 사용자가 커스텀 시작일을 직접 입력해 기준선을 생성할 수도 있다 (두 방식 모두 지원).
- Q: 페이지 진입 시 기본 화면 구성은? → A: 진입 즉시 "현재 진행 중 이벤트 vs 가장 유사한 과거 이벤트" 비교 차트를 자동 표시. 사용자는 추가 선택 없이 드래그만으로 탐색.
- Q: 모바일 조작 방식은? → A: 좌우 스와이프만으로 모든 컨트롤 가능. 별도 버튼·탭 조작 없이 스와이프 제스처로 이벤트 전환·차트 탐색·지표 변경 모두 처리.

## User Scenarios & Testing

### User Story 1 - 위기 이벤트 선택 및 지표 움직임 조회 (Priority: P1)

사용자가 "코로나19 팬데믹", "2008 금융위기", "러시아-우크라이나 전쟁" 등 기록된 위기 이벤트 목록에서 하나를 선택하면, 해당 이벤트 발생 전후(예: -30일 ~ +180일)의 주요 시장 지표 변화를 차트로 확인할 수 있다.

**Why this priority**: 핵심 기능. 이것만 동작해도 즉시 실용적 가치를 제공한다.

**Independent Test**: 위기 이벤트 목록에서 "2008 금융위기"를 선택하면, S&P500·코스피·금·달러인덱스 등 주요 지표의 기간별 등락 차트가 표시되는지 확인한다.

**Acceptance Scenarios**:

1. **Given** 앱에 접속한 상태, **When** 위기 이벤트 목록에서 "2008 금융위기"를 선택, **Then** 이벤트 발생일 기준 전후 기간 동안의 주요 지표 움직임이 차트로 표시된다.
2. **Given** 위기 이벤트 상세 화면, **When** 지표 탭 간 전환(주식/채권/원자재/환율), **Then** 해당 카테고리의 지표 데이터가 즉시 표시된다.
3. **Given** 위기 이벤트 상세 화면, **When** 조회 기간을 변경(이벤트 기준 ±30일 / ±90일 / ±180일), **Then** 선택된 기간에 맞는 차트가 업데이트된다.
4. **Given** 위기 이벤트 상세 화면, **When** 차트를 확인, **Then** 이벤트 발생일이 차트에 수직선으로 표시되어 구분된다.

---

### User Story 2 - 위기 유형별 필터링 및 이벤트 요약 통계 (Priority: P2)

사용자는 위기 유형(전쟁, 팬데믹, 금융위기, 자연재해 등)으로 이벤트를 필터링하고, 선택한 이벤트에서 각 지표의 최대 낙폭·최대 상승폭·회복 소요 기간 등 핵심 통계를 요약 카드로 확인할 수 있다.

**Why this priority**: 차트만으로는 파악하기 어려운 핵심 수치를 빠르게 제공하여 의사결정을 돕는다.

**Independent Test**: 이벤트 목록에서 "전쟁" 필터를 적용하면 관련 이벤트만 표시되고, 이벤트 선택 시 S&P500의 최대 낙폭(-X%)과 회복 소요일이 요약 카드로 표시된다.

**Acceptance Scenarios**:

1. **Given** 이벤트 목록 화면, **When** "팬데믹" 유형 필터 선택, **Then** 팬데믹 관련 이벤트만 목록에 표시된다.
2. **Given** 위기 이벤트 상세 화면, **When** 지표 요약 카드 확인, **Then** 해당 이벤트 기간 중 각 지표의 최대 낙폭(MDD), 최대 상승폭, 회복 소요 기간이 표시된다.
3. **Given** 위기 이벤트 상세 화면, **When** 이벤트 설명 확인, **Then** 이벤트 발생 배경, 기간, 경제적 영향을 요약한 텍스트가 표시된다.

---

### User Story 3 - 현재 상황 vs 과거 이벤트 비교 차트 (Priority: P3)

페이지 진입 시 "현재 진행 중인 이벤트 vs 가장 유사한 과거 이벤트" 비교 차트가 자동으로 표시된다. 사용자는 추가 선택 없이 드래그만으로 "이란-미국 갈등(현재) vs 1973년 오일쇼크"처럼 현재와 과거를 나란히 비교할 수 있다. 최대 3개 이벤트(진행 중 포함)를 비교할 수 있으며, 목록에 없는 사건은 커스텀 시작일을 입력해 기준선으로 설정할 수 있다.

**Why this priority**: 단순 조회 이상의 패턴 인사이트를 제공하나, P1·P2가 선행되어야 한다. 현재 진행 중인 위기와 과거 패턴을 즉시 대조할 수 있어 실용적 가치가 크다.

**Independent Test**: 페이지 진입 시 현재 진행 중인 이벤트와 가장 유사한 과거 이벤트의 비교 차트가 자동 표시되고, 사용자가 차트를 드래그해 기간을 탐색할 수 있다.

**Acceptance Scenarios**:

1. **Given** 위기 히스토리 페이지 진입, **When** 아무 조작 없이 페이지 로드 완료, **Then** 현재 진행 중인 이벤트와 가장 유사한 과거 이벤트의 비교 차트가 자동으로 표시된다.
2. **Given** 비교 차트, **When** 차트 드래그로 기간 탐색, **Then** 두 이벤트의 주요 지표 변화가 이벤트 발생일(0일) 기준 상대 변화율(%)로 정렬되어 나란히 표시된다.
3. **Given** 이벤트 목록, **When** 이벤트 2~3개 선택 (진행중 이벤트 포함 가능), **Then** 선택된 이벤트들의 특정 지표 변화를 겹쳐 비교하는 차트가 표시된다.
4. **Given** 비교 차트, **When** 특정 시점에 마우스 오버/터치, **Then** 각 이벤트의 해당 시점 지표 값과 변화율이 툴팁으로 표시된다.
5. **Given** 목록에 없는 현재 사건 비교 필요, **When** 커스텀 시작일 입력, **Then** 해당 날짜를 Day 0으로 하는 현재 시장 데이터가 비교 차트에 기준선으로 추가된다.

---

### Edge Cases

- 데이터가 없는 시기(해당 시장이 존재하지 않던 지표)는 "데이터 없음"으로 명확히 표시
- 이벤트 발생일로부터 조회 기간이 현재 날짜를 초과하는 경우(최근 이벤트) 가용 데이터까지만 표시
- 복수 이벤트 비교 시 최대 3개 초과 선택 시도 시 "최대 3개까지 선택 가능합니다" 안내
- 데이터 로딩 실패 시 실패 메시지와 함께 재시도 버튼 표시
- 이벤트 기간 중 시장 휴장일(주말·공휴일) 데이터는 이전 영업일 값으로 표시

## Requirements

### Functional Requirements

- **FR-001**: 시스템은 글로벌 위기 이벤트 목록(최소 15개)을 제공해야 하며, 전쟁·팬데믹·금융위기·자연재해 카테고리를 포함해야 한다
- **FR-002**: 각 이벤트에 대해 발생일 기준 전후 최소 ±180일의 주요 시장 지표 일별 데이터를 표시해야 한다
- **FR-003**: 제공 지표는 최소 8개로 한다: S&P500, 코스피, 나스닥, 금(현물), 원유(WTI), 달러인덱스(DXY), 미국채10년금리, 원/달러 환율
- **FR-004**: 사용자는 조회 기간을 이벤트 기준 ±30일 / ±90일 / ±180일 중 선택할 수 있어야 한다
- **FR-005**: 위기 유형별 필터(전쟁, 팬데믹, 금융위기, 기타)로 이벤트 목록을 필터링할 수 있어야 한다
- **FR-006**: 각 이벤트에 대해 이벤트 발생 배경 및 주요 특징을 간략히 설명하는 텍스트를 제공해야 한다
- **FR-007**: 이벤트-지표별 요약 통계(최대 낙폭, 최대 상승폭, 회복 소요 기간)를 표시해야 한다
- **FR-008**: 차트에 이벤트 발생일을 수직선으로 표시하여 기준점을 명확히 해야 한다
- **FR-009**: 동일 유형의 이벤트 최대 3개를 선택하여 특정 지표의 변화를 이벤트 발생일 기준 상대 변화율(%)로 겹쳐 비교하는 차트를 제공해야 한다
- **FR-010**: 모바일 화면에서 좌우 스와이프만으로 모든 조작(이벤트 전환, 차트 기간 탐색, 지표 변경)이 가능해야 한다. 별도 버튼·탭 조작 없이 스와이프 제스처만으로 핵심 기능 전체를 컨트롤할 수 있어야 한다
- **FR-011**: 로그인 없이 모든 사용자가 조회할 수 있어야 한다
- **FR-012**: 큐레이션 이벤트 목록에 현재 진행 중인 이벤트를 "진행중" 배지로 구분 포함해야 하며, 사용자가 임의 시작일을 입력해 커스텀 기준선으로 비교 차트에 추가할 수 있어야 한다
- **FR-013**: 위기 히스토리 페이지 진입 시 추가 선택 없이 즉시 비교 차트(현재 진행 중 이벤트 vs 가장 유사한 과거 이벤트)가 자동 표시되어야 하며, 사용자 주요 조작은 차트 드래그(기간 탐색)로 최소화한다

### Key Entities

- **CrisisEvent**: 위기 이벤트 (id, 이름, 유형, 발생일, 종료일, 설명, 심각도 등급, is_ongoing: 진행 중 여부)
- **MarketIndicator**: 시장 지표 (id, 이름, 카테고리, 단위, 데이터 소스 식별자)
- **IndicatorDataPoint**: 지표 일별 데이터 (indicator_id, date, value, change_pct_from_event_start)
- **EventIndicatorStats**: 이벤트-지표별 요약 통계 (event_id, indicator_id, max_drawdown_pct, max_gain_pct, recovery_days)

## Success Criteria

### Measurable Outcomes

- **SC-001**: 위기 이벤트 선택 후 3초 이내에 주요 지표 차트가 표시된다
- **SC-002**: 최소 15개 위기 이벤트, 8개 이상 시장 지표 데이터를 제공한다
- **SC-003**: 사용자가 이벤트 선택에서 지표 차트 확인까지 2단계 이내로 완료 가능하다
- **SC-004**: 모바일 화면에서 좌우 스와이프만으로 이벤트 전환·차트 탐색·지표 변경을 모두 완료할 수 있다 (버튼 클릭 없이)
- **SC-005**: 복수 이벤트 비교 차트에서 각 이벤트의 지표 선이 명확히 구분되어 색상으로 표시된다
- **SC-006**: 이벤트별 요약 통계(최대 낙폭 등)가 차트와 동일 화면에서 확인 가능하다
- **SC-007**: 페이지 진입 후 추가 클릭 없이 3초 이내에 "현재 진행 중 이벤트 vs 과거 유사 이벤트" 비교 차트가 자동 표시된다
- **SC-008**: 비교 차트에서 커스텀 시작일 입력 후 2초 이내에 해당 날짜 기준 현재 시장 데이터가 기준선으로 추가된다

## Assumptions

- 시장 지표 데이터는 공개 데이터 소스(예: Yahoo Finance)에서 획득 가능하다고 가정
- 이벤트 목록과 설명 텍스트는 초기에 정적 데이터로 관리하며, 추후 관리자 편집 기능 추가 가능
- 조회 가능 데이터 범위는 1950년 이후로 제한 (이전 데이터는 신뢰성 낮음)
- 비교 차트의 기준은 이벤트 발생일의 지표 값을 0%로 설정한 상대 변화율로 통일
- 로그인 불필요 — 모든 방문자가 인증 없이 조회 가능

# Feature Specification: Toss/Domino UI Redesign

**Feature Branch**: `014-toss-ui-redesign`
**Created**: 2026-04-03
**Status**: Draft
**Input**: 도미노, 토스의 디자인 패턴과 가독성있는 ui(색상, 폰트, 폰트사이즈, 차트가독성, 필요한정보의 나열방식) 밴치마킹

## Overview

현재 앱의 UI를 토스증권/도미노 스타일로 리디자인한다. 핵심 목표는 **다크 테마 + 고대비 색상 + 간결한 정보 계층** 이다. 토스증권은 순수 블랙 배경(`#000000`), 진한 회색 카드(`#1c1c1e`), 상승 핑크-레드(`#ff4b6a`), 하락 파란(`#4285f4`)의 조합으로 야간 환경에서의 가독성을 극대화한다. 폰트 크기와 정보 나열 방식(레이블-왼쪽/값-오른쪽)도 동일 패턴을 따른다.

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 다크 컬러 시스템 적용 (Priority: P1)

사용자가 대시보드, 관심종목, 스캔 결과 화면을 열었을 때 배경은 순수 블랙, 카드는 진한 회색, 상승/하락 색상이 토스증권 스타일로 표시된다.

**Why this priority**: 색상 시스템은 모든 컴포넌트의 기반이며, 이를 먼저 확립해야 폰트/레이아웃 작업이 일관성을 유지할 수 있다.

**Independent Test**: 대시보드 화면 하나만 보아도 다크 배경 + 핑크 상승 + 파랑 하락 조합이 즉시 확인된다.

**Acceptance Scenarios**:

1. **Given** 앱을 처음 열었을 때, **When** 대시보드가 로드되면, **Then** 배경은 `#000000`, 카드 배경은 `#1c1c1e`, 상승 수치는 `#ff4b6a`, 하락 수치는 `#4285f4`로 표시된다.
2. **Given** 관심종목 카드를 보았을 때, **When** BUY 신호 종목이 있으면, **Then** 강조색(상승)이 핑크-레드 계열로 표시된다.
3. **Given** 모든 텍스트를 보았을 때, **When** 기본 텍스트를 확인하면, **Then** 주 텍스트는 흰색(`#ffffff`), 보조 텍스트는 회색(`#8e8e93`)으로 구분된다.

---

### User Story 2 - 타이포그래피 스케일 정비 (Priority: P1)

사용자가 각 화면의 수치, 레이블, 섹션 제목을 볼 때 크기·굵기가 토스증권처럼 명확한 계층을 이룬다.

**Why this priority**: 색상 시스템과 동시에 적용되어야 전체적인 가독성이 완성된다.

**Independent Test**: 종목 카드 하나에서 종목명(굵음 큰 글씨), 현재가(숫자 강조), 등락률(작은 보조)이 세 단계 계층으로 읽힌다.

**Acceptance Scenarios**:

1. **Given** 종목 카드를 보았을 때, **When** 레이아웃을 확인하면, **Then** 종목명 ≥ 15px 굵음, 현재가 ≥ 18px 숫자, 등락률 ≤ 13px 보조로 크기가 구분된다.
2. **Given** 섹션 헤더를 보았을 때, **When** 제목을 읽으면, **Then** 섹션 제목은 16px 이상 semi-bold 이다.
3. **Given** 모바일 화면(360px 기준)에서, **When** 전체 화면을 보면, **Then** 텍스트가 잘리거나 줄바꿈 없이 모두 읽힌다.

---

### User Story 3 - 카드 레이아웃 정보 나열 방식 개선 (Priority: P2)

사용자가 종목 카드나 지표 행을 볼 때 레이블은 왼쪽, 값은 오른쪽으로 정렬되어 빠른 스캔이 가능하다.

**Why this priority**: 색상/폰트 기반 위에 정보 구조를 개선하면 가독성이 더 높아진다.

**Independent Test**: 관심종목 카드 하나에서 레이블-왼쪽/값-오른쪽 패턴이 일관되게 보인다.

**Acceptance Scenarios**:

1. **Given** 종목 카드를 보았을 때, **When** 정보 항목(거래량, 시가 등)을 확인하면, **Then** 레이블은 좌측 회색, 값은 우측 흰색으로 정렬된다.
2. **Given** 시장지표 패널을 보았을 때, **When** VIX·코스피·S&P500 행을 확인하면, **Then** 지표명-왼쪽, 수치+등락-오른쪽 배열이 일관된다.
3. **Given** 설정 화면을 보았을 때, **When** 각 설정 항목을 보면, **Then** 동일한 레이블-왼쪽/컨트롤-오른쪽 패턴이 유지된다.

---

### User Story 4 - 차트 가독성 개선 (Priority: P2)

사용자가 차트를 볼 때 격자선이 최소화되고, 상승/하락 색상이 다크 배경에 최적화되며, 레이블이 겹치지 않는다.

**Why this priority**: 차트는 핵심 판단 도구이므로 다크 테마에 맞는 색상 조정이 필요하다.

**Independent Test**: 차트 화면에서 캔들/라인이 어두운 배경에서 뚜렷이 구분되고 격자가 눈에 띄지 않는다.

**Acceptance Scenarios**:

1. **Given** 차트 화면을 열었을 때, **When** 캔들차트를 보면, **Then** 상승 캔들은 핑크-레드(`#ff4b6a`), 하락 캔들은 파란(`#4285f4`) 계열로 표시된다.
2. **Given** 차트 배경을 확인할 때, **When** 격자선을 보면, **Then** 격자선 투명도가 15% 이하로 배경에 묻히지 않는 수준이다.
3. **Given** 모바일에서 차트를 볼 때, **When** 가격 레이블을 확인하면, **Then** Y축 레이블이 캔들과 겹치지 않는다.

---

### User Story 5 - 하단 탭바 토스 스타일 개선 (Priority: P3)

모바일 하단 탭바가 토스증권처럼 아이콘+텍스트 조합으로 명확하고, 활성 탭이 핑크-레드로 강조된다.

**Why this priority**: 네비게이션은 이미 동작하므로 시각적 완성도를 높이는 마지막 단계이다.

**Independent Test**: 모바일에서 탭을 눌러 활성 탭 색상이 토스 스타일로 변경되는 것을 확인한다.

**Acceptance Scenarios**:

1. **Given** 모바일 하단 탭바를 보았을 때, **When** 활성 탭을 확인하면, **Then** 아이콘과 텍스트가 핑크-레드(`#ff4b6a`)로 강조된다.
2. **Given** 비활성 탭을 보았을 때, **When** 색상을 확인하면, **Then** 비활성 탭은 중간 회색(`#636366`)으로 표시된다.
3. **Given** 탭바 배경을 보았을 때, **When** 투명도를 확인하면, **Then** 탭바는 반투명(`rgba(0,0,0,0.85)`) + backdrop-blur 처리된다.

---

### Edge Cases

- 현재 라이트 모드 설정이 있다면 다크 모드가 강제 적용되어야 한다 (시스템 모드 무시).
- 색상 대비가 WCAG AA 기준(4.5:1)을 충족해야 접근성을 해치지 않는다.
- 숫자가 매우 길 때(예: 1,234,567원) 카드 레이아웃이 깨지지 않아야 한다.
- 차트 데이터가 없을 때 빈 상태 화면도 다크 테마를 유지해야 한다.

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 시스템은 전체 배경색을 `#000000`(순수 블랙)으로 적용해야 한다.
- **FR-002**: 카드 및 모달 배경은 `#1c1c1e`(진한 회색)를 사용해야 한다.
- **FR-003**: 상승(양수) 수치는 `#ff4b6a` 색상으로, 하락(음수) 수치는 `#4285f4` 색상으로 표시해야 한다.
- **FR-004**: 주요 텍스트는 `#ffffff`, 보조 텍스트는 `#8e8e93`으로 색상 계층을 구분해야 한다.
- **FR-005**: 섹션 구분선은 `#2c2c2e`(매우 어두운 회색) 단일 라인으로 적용해야 한다.
- **FR-006**: 종목명은 15px 이상 font-weight 600, 현재가는 18px 이상 font-weight 700, 보조 정보는 12~13px font-weight 400으로 타이포그래피 계층을 적용해야 한다.
- **FR-007**: 모든 정보 행(지표, 카드 세부사항, 설정 항목)은 레이블-왼쪽/값-오른쪽 패턴을 따라야 한다.
- **FR-008**: 차트 캔들 상승색은 `#ff4b6a`, 하락색은 `#4285f4`로 변경해야 한다.
- **FR-009**: 차트 격자선 투명도는 10~15% 수준으로 설정해야 한다.
- **FR-010**: 카드 컴포넌트는 border-radius 12px 이상, 내부 padding 16px 이상을 유지해야 한다.
- **FR-011**: 모바일 하단 탭바의 활성 탭 색상은 `#ff4b6a`로 강조되어야 한다.
- **FR-012**: 탭바 배경은 `rgba(0,0,0,0.85)` + backdrop-blur로 반투명 처리해야 한다.
- **FR-013**: UI 리디자인 작업은 순수 시각적 변경에 한정되며, 관심종목 추가/삭제, 스캔, 차트 조회, 설정, 텔레그램 알림 등 기존 기능은 하나도 제거되거나 비활성화되어서는 안 된다.
- **FR-014**: 모바일에서 대시보드 내 섹션 간 이동은 상하 스와이프(수직 스냅 스크롤) 방식을 유지한다. 하단 탭바는 제거하지 않고 페이지 네비게이션 수단으로 유지한다.
- **FR-015**: 화면 내에서 탭 컴포넌트(예: 1개월/3개월/1년 기간 탭, 연간/분기 탭)를 사용하여 정보를 나누는 것은 허용된다.

### Key Entities *(include if feature involves data)*

- **ColorSystem**: 앱 전체에 사용되는 색상 토큰 (배경, 카드, 상승, 하락, 텍스트 계층)
- **TypographyScale**: 폰트 크기·굵기 계층 (제목, 본문, 보조, 숫자)
- **CardPattern**: 종목 카드의 레이아웃 패턴 (레이블-왼쪽/값-오른쪽, radius, padding)

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 대시보드 화면을 처음 본 사용자가 상승/하락 구분을 5초 이내에 파악할 수 있다.
- **SC-002**: 텍스트 색상 대비율이 WCAG AA 기준(4.5:1)을 모든 주요 텍스트에서 충족한다.
- **SC-003**: 모바일 360px 너비에서 모든 카드 텍스트가 잘리거나 줄바꿈 없이 표시된다.
- **SC-004**: 차트 배경 대비 캔들 색상이 즉시 구분 가능한 수준(대비율 3:1 이상)이다.
- **SC-005**: 기존 모든 기능(관심종목 추가/삭제/수정, 스캔 실행, 차트 조회, 설정 변경, 텔레그램 알림)이 UI 변경 후에도 동일하게 정상 동작한다 — 어떠한 기능도 제거·비활성화되지 않는다.
- **SC-006**: 다크 테마가 브라우저/OS 라이트 모드 설정에 무관하게 항상 적용된다.

## Assumptions

- 이 기능은 기존 Tailwind CSS 유틸리티 클래스를 커스터마이징하거나 인라인 스타일로 색상을 재정의한다.
- 다국어(i18n) 지원은 범위 밖이며 한국어 UI를 기준으로 한다.
- 라이트 모드 토글 기능은 추가하지 않고 다크 모드만 지원한다.
- 차트 라이브러리(lightweight-charts v5)의 색상 옵션을 활용하여 캔들 색상을 재정의한다.

- Q: 모바일 섹션 이동 방식 및 네비게이션 구조는? → A: 상하 스와이프(수직 스냅 스크롤)로 유지, 하단 탭바 네비게이션 유지, 화면 내 탭 컴포넌트 허용.

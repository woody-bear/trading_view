# Quickstart: Toss UI Redesign 검증 시나리오 (014)

## 전제 조건

```bash
cd frontend && pnpm dev  # localhost:3000
```

---

## 시나리오 1: 컬러 시스템 검증

1. 브라우저에서 `http://localhost:3000` 열기
2. 확인 항목:
   - [ ] 페이지 배경이 순수 블랙(`#000000`)
   - [ ] 카드 배경이 진한 회색(`#1c1c1e`)
   - [ ] 상승 수치(양수 등락률)가 핑크-레드(`#ff4b6a`)
   - [ ] 하락 수치(음수 등락률)가 파란(`#4285f4`)
   - [ ] 종목명은 흰색, 레이블은 회색(`#8e8e93`)으로 구분

---

## 시나리오 2: 타이포그래피 계층 검증

1. 관심종목 카드 하나를 확인
2. 개발자 도구(F12) → Elements로 폰트 크기 확인:
   - [ ] 종목명: ≥15px, font-weight 600
   - [ ] 현재가: ≥18px, font-weight 700, font-family mono
   - [ ] 등락률: ≤13px, font-weight 600, mono
3. 모바일 뷰(360px)로 전환:
   - [ ] 텍스트 잘림이나 줄바꿈 없음

---

## 시나리오 3: 차트 가독성 검증

1. 관심종목 클릭 → 종목 상세(차트) 화면
2. 확인 항목:
   - [ ] 차트 배경이 블랙
   - [ ] 상승 캔들: 핑크-레드
   - [ ] 하락 캔들: 파란
   - [ ] 수평 격자선이 희미하게만 보임(15% 이하 투명도)
   - [ ] Y축 레이블 캔들과 겹치지 않음

---

## 시나리오 4: 하단 탭바 검증 (모바일)

1. 브라우저 모바일 뷰(375px) 또는 실제 모바일
2. 확인 항목:
   - [ ] 활성 탭 아이콘+텍스트가 `#ff4b6a` 핑크-레드
   - [ ] 비활성 탭이 `#636366` 회색
   - [ ] 탭바 배경이 반투명 블랙 + blur
3. 각 탭 클릭 → 해당 페이지로 정상 이동 확인

---

## 시나리오 5: 기존 기능 보존 검증

1. 관심종목 추가 → 정상 추가됨
2. 스캔 실행 → 결과 표시됨
3. 설정 → 텔레그램 설정 저장됨
4. 차트 기간 탭(1개월/3개월/1년) 전환 → 데이터 변경됨
5. Fear&Greed 게이지 → 수치 표시됨

---

## 접근성 확인 (선택)

크롬 개발자도구 → Lighthouse → Accessibility:
- 주요 텍스트(흰색 on 검정): 대비율 ≥ 4.5:1 (WCAG AA)
- 상승색(`#ff4b6a` on `#000000`): 대비율 ≥ 3:1 확인

# Feature Specification: BUY 차트 라벨 클릭 — 사례 스크랩 추가

**Feature Branch**: `010-chart-buy-scrap`
**Created**: 2026-03-31
**Status**: Draft
**Input**: "buy차트 라벨위에 마우스를 올리면 클릭해서 BUY사례스크랩에 사례를 분석해서 추가하는 기능"

**목적**: 과거 매수 사례를 수집하여 "좋은 매수"가 발생했던 지표 조건(RSI·스퀴즈레벨·MACD 등)을 사용자가 직접 검토·메모하며 패턴을 파악하고, 이를 통해 향후 BUY 조건 종목 찾기를 강화한다. 집계 자동화가 아닌 사용자의 능동적 학습이 핵심이다.

## User Scenarios & Testing *(mandatory)*

### User Story 1 — BUY 라벨 호버 → 스크랩 버튼 → 자동 분석 저장 (Priority: P1)

사용자가 종목 상세 차트에서 BUY 신호 라벨 위에 마우스를 올리면 "이 사례 저장" 버튼이 나타난다. 버튼을 클릭하면 해당 BUY 시점의 기술적 지표값이 자동 수집되어 BUY 사례 스크랩에 추가된다.

**Why this priority**: 전체 기능의 진입점. 이것이 없으면 나머지 기능을 사용할 수 없다.

**Independent Test**: 차트 BUY 라벨 호버 → 저장 버튼 클릭 → 스크랩 목록에서 해당 종목/날짜 항목 확인.

**Acceptance Scenarios**:

1. **Given** 종목 상세 차트에 BUY 라벨이 표시된 상태, **When** 사용자가 BUY 라벨 위에 마우스를 올리면, **Then** "이 BUY 사례 저장" 버튼이 즉시 표시된다.
2. **Given** 저장 버튼이 표시된 상태, **When** 사용자가 버튼을 클릭하면, **Then** 해당 날짜의 RSI·BB·MACD·거래량비율·스퀴즈레벨이 자동 수집되어 사례로 저장된다.
3. **Given** 이미 스크랩된 BUY 라벨 위에 마우스를 올리면, **Then** 버튼이 "저장됨" 상태로 표시되어 중복 저장을 방지한다.
4. **Given** 마우스가 BUY 라벨 영역을 벗어나면, **Then** 버튼은 사라진다.

---

### User Story 2 — 스크랩 목록 조회 및 삭제 (Priority: P2)

사용자가 스크랩된 BUY 사례 목록을 확인하고, 저장된 지표값과 메모를 직접 검토하며 "어떤 조건에서 좋은 매수가 발생했는지" 패턴을 파악할 수 있다. 불필요한 사례는 삭제할 수 있다. 집계·통계 자동 산출은 제공하지 않으며, 패턴 파악은 사용자가 직접 수행한다.

**Why this priority**: 저장 후 활용이 가능해야 기능의 가치가 완성된다. 사례를 보며 BUY 조건 강화의 근거를 직접 도출하는 것이 이 기능의 핵심 목적이다.

**Independent Test**: 스크랩 목록 페이지 진입 → 저장된 사례 확인 → 상세 지표 확인 → 삭제 후 목록에서 제거 확인.

**Acceptance Scenarios**:

1. **Given** BUY 사례가 1개 이상 저장된 상태, **When** 사용자가 스크랩 목록을 열면, **Then** 종목명·신호일·RSI·스퀴즈레벨이 목록에 표시된다.
2. **Given** 스크랩 목록, **When** 사용자가 특정 사례를 클릭하면, **Then** 해당 종목 차트로 이동하며 해당 BUY 날짜가 강조 표시된다.
3. **Given** 스크랩 목록, **When** 사용자가 특정 사례를 삭제하면, **Then** 목록에서 제거되고 차트 BUY 라벨의 "저장됨" 표시도 해제된다.

---

### User Story 3 — 사례에 메모 추가 (Priority: P3)

사용자가 스크랩한 BUY 사례에 매수 이유, 판단 근거 등 자유 메모를 작성하여 향후 학습에 활용한다.

**Why this priority**: 자동 분석만으로도 충분히 활용 가능하므로 메모는 부가 기능이다.

**Independent Test**: 스크랩 목록에서 사례 선택 → 메모 입력 → 저장 → 재방문 시 메모 유지 확인.

**Acceptance Scenarios**:

1. **Given** 스크랩된 사례, **When** 사용자가 메모 영역을 클릭하면, **Then** 텍스트 입력창이 활성화된다.
2. **Given** 메모 입력 후 저장, **When** 페이지를 재방문하면, **Then** 메모 내용이 유지된다.

---

### Edge Cases

- BUY 라벨이 없는 종목(신호 미발생)에서는 스크랩 버튼이 나타나지 않는다.
- 모바일(터치) 환경에서는 호버 대신 라벨 탭으로 버튼을 표시한다.
- 차트 데이터 로딩 전에는 스크랩 버튼 인터랙션이 비활성화된다.
- 동일 종목·동일 BUY 날짜를 중복 저장하려 할 때 경고 메시지를 표시하고 저장하지 않는다.
- 지표값을 불러오지 못하면 저장은 가능하되 해당 지표 필드는 비어있음으로 처리된다.

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 차트에 표시된 BUY 신호 마커 위에 마우스를 올리면 DOM 오버레이 방식으로 저장 버튼이 표시되어야 한다. 버튼 위치는 마커 좌표를 차트 API로 변환하여 절대 위치 CSS로 배치한다. 모바일(터치) 환경에서는 마커 탭으로 동일하게 표시한다.
- **FR-002**: 저장 버튼 클릭 시 서버에 해당 종목·날짜의 지표값(RSI, BB %B, BB 폭, MACD 히스토그램, 거래량비율, 스퀴즈레벨, EMA 정배열 여부)을 재요청하여 수집하고, 수집된 값과 함께 사례를 저장해야 한다. 서버 요청 실패 시 해당 지표 필드는 빈값으로 저장 가능하다.
- **FR-003**: 이미 스크랩된 BUY 날짜의 라벨에는 "저장됨" 시각적 표시(색상·아이콘 변경)가 나타나야 한다.
- **FR-004**: 사례 목록은 단일 열 리스트로 표시되며(그리드 다열 배치 사용 안 함), 각 항목은 종목명, 종목코드, 신호일, RSI, 스퀴즈레벨, 저장 일시를 표시해야 한다.
- **FR-005**: 사례 항목은 아코디언 방식으로 펼쳐지며, 펼쳐진 내부에 "차트 보기" 버튼을 통해 해당 종목 차트 페이지로 이동할 수 있어야 한다. 아코디언 헤더 클릭은 펼치기/접기로 유지한다.
- **FR-006**: 사례 항목 삭제 시 인라인 확인 UI(확인/취소 버튼)를 표시하며, 확인 후 삭제되어야 한다. 삭제 완료 시 차트 BUY 라벨의 "저장됨" 표시도 함께 해제되어야 한다. window.confirm은 사용하지 않는다.
- **FR-007**: 아코디언 펼쳐진 내부에 메모 textarea가 인라인으로 표시되며, 입력 중단 후 1~2초 디바운스로 자동 저장되어야 한다. 저장 중에는 "저장 중..." 표시, 완료 시 "저장됨" 표시가 나타난다. 별도 저장 버튼은 표시하지 않는다.
- **FR-008**: 동일 종목·동일 BUY 날짜를 중복 저장하려 할 때 경고 메시지를 표시해야 한다.
- **FR-009**: 스크랩 목록 각 항목에 출처 뱃지를 표시해야 한다. 차트 BUY 라벨에서 자동 저장된 사례는 "📊 차트", 수동으로 추가한 사례는 "✏️ 수동"으로 구분 표시한다. 출처별 필터링은 제공하지 않는다.

### Key Entities

- **BUY 사례 (PatternCase)**: 스크랩된 BUY 신호 사례. 종목코드, 종목명, 시장, 신호일, 저장 당시 지표값(RSI·BB·MACD·거래량비율·스퀴즈레벨·EMA 정배열 여부), 사용자 메모, 저장 일시 포함.
- **BUY 라벨**: 차트 위에 표시된 BUY 신호 마커. 날짜 정보와 스크랩 여부(저장됨/미저장) 상태를 가진다.

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: BUY 라벨 호버 후 저장 버튼이 0.3초 이내에 표시된다.
- **SC-002**: 저장 버튼 클릭 후 사례가 3초 이내에 스크랩 목록에 반영된다.
- **SC-003**: 사례 저장 성공 시 즉각적인 시각적 피드백(라벨 색상 변경 또는 토스트 메시지)이 표시된다.
- **SC-004**: 스크랩 목록에서 조회, 삭제, 메모 저장이 모두 별도 페이지 이동 없이 가능하다.
- **SC-005**: 모바일 터치 환경에서도 동일하게 저장·조회·삭제가 가능하다.

## Assumptions

- BUY 라벨은 기존 종목 상세 차트 화면에 이미 표시되어 있으며, 날짜 정보를 포함하고 있다.
- 지표값은 차트에 이미 로드된 데이터에서 추출 가능하다.
- PatternCase 데이터 모델은 기존 코드베이스에 이미 추가되어 있다.
- 스크랩 목록 페이지(Scrap.tsx)는 기본 구조가 이미 존재한다.
- 로그인한 사용자의 사례만 저장·조회되며, 미로그인 상태에서는 저장 버튼이 비활성화된다.

## Out of Scope

- AI/ML 기반 자동 수익률 예측 또는 패턴 추천
- 사례 공유 기능 (다른 사용자와 공유)
- 자동 백테스팅 또는 과거 수익률 자동 계산
- 저장 사례들의 지표 분포·평균값 등 집계 요약 뷰 (패턴 파악은 사용자가 직접 수행)

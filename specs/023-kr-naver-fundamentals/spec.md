# Feature Specification: KR 개별주 PER/PBR/EPS/BPS 네이버 파이낸스 보강

**Feature Branch**: `023-kr-naver-fundamentals`
**Created**: 2026-04-15
**Status**: Draft
**Input**: User description: "KR 개별주 PER/PBR/EPS/BPS 네이버 파이낸스 크롤러 보강"

## Background

현재 KR 개별 상장주식의 가치 지표(PER/PBR/EPS/BPS)는 주 데이터 소스(yfinance)에서 **주요 대형주 상당수가 결측(`None`)**으로 반환되어 사용자에게 `—`로 노출되고 있다. 예: SK하이닉스·삼성전자 등 핵심 종목에서 이 네 항목이 비어 있다.

경쟁 서비스(토스증권·네이버 증권 등)는 같은 종목에서 정확한 값을 제공하므로, 본 기능은 **국내 상장주에 한해 보조 데이터 소스(네이버 파이낸스)로 결측을 보강**하여 사용자 가치 분석 경험을 토스·네이버 수준으로 끌어올리는 것을 목표로 한다.

## User Scenarios & Testing *(mandatory)*

### User Story 1 — 주요 KR 대형주의 가치 지표 결측 없이 확인 (Priority: P1)

가치 분석 탭을 여는 사용자가 SK하이닉스·삼성전자·LG에너지솔루션 등 주요 KR 대형주에서 PER/PBR/EPS/BPS 수치를 **`—`가 아닌 실제 값**으로 볼 수 있다.

**Why this priority**: 본 기능의 핵심 가치. 현재 주요 대형주에서 네 지표가 모두 비어 가치 분석의 실질적 유용성이 크게 훼손되어 있음. 이 스토리 하나만 구현되어도 MVP 가치 성립.

**Independent Test**: KR 대형주 샘플 10종목(예: 005930·000660·051910·035420·005380·000270·068270·207940·035720·006400)에 대해 가치 분석 탭의 PER/PBR/EPS/BPS 중 **최소 3개**가 결측 없이 표시됨을 확인.

**Acceptance Scenarios**:

1. **Given** SK하이닉스(000660) 상세 진입, **When** 가치 분석 탭 활성화, **Then** PER/PBR/EPS/BPS 4개 카드 모두 실제 값 노출 (예: PER 18.9배, PBR 6.4배, EPS 58,954원, BPS 174,538원).
2. **Given** 삼성전자(005930) 상세 진입, **When** 가치 분석 탭 활성화, **Then** 네 지표 모두 실제 값 노출.
3. **Given** 네이버 파이낸스가 특정 지표를 제공하지 않는 종목, **When** 가치 분석 탭 활성화, **Then** 제공되지 않는 개별 항목만 `—`, 나머지는 정상 표시.

---

### User Story 2 — 보강 실패 시에도 기존 경험 유지 (Priority: P1)

네이버 파이낸스 접속·파싱·일시적 응답 실패 시 **기존 yfinance 값으로 폴백**하여 사용자 경험이 악화되지 않도록 한다.

**Why this priority**: 외부 크롤링은 본질적으로 불안정. 실패가 서비스 장애로 전파되면 안 됨.

**Independent Test**: 네이버 파이낸스 응답을 강제로 실패시킨 상태에서 기존 yfinance 응답만으로 가치 분석 탭이 정상 렌더(결측은 `—`, 값이 있던 ROE 등은 그대로)됨을 확인.

**Acceptance Scenarios**:

1. **Given** 네이버 파이낸스 요청 타임아웃, **When** 가치 분석 탭 활성화, **Then** yfinance 값 기반으로 카드가 표시되며 에러 메시지 없음.
2. **Given** 네이버 응답 HTML 구조가 변경되어 파싱 실패, **When** 가치 분석 탭 활성화, **Then** 해당 지표만 결측(`—`), 다른 지표 정상 표시.
3. **Given** 네이버 크롤링이 반복 실패하는 종목, **When** 재진입, **Then** 매 요청마다 네이버에 재시도 폭주 없이 적절히 억제된다.

---

### User Story 3 — 네이버·yfinance 값이 둘 다 있을 때 출처 명확화 (Priority: P2)

사용자가 카드에 표시된 수치의 **기준 출처/시점**을 식별할 수 있어 값 차이에 대한 혼란이 없도록 한다.

**Why this priority**: 데이터 소스가 두 개로 늘어나면 "왜 토스와 우리 앱 값이 다르지?" 같은 의문이 발생. 최소한의 출처 표기가 필요.

**Independent Test**: 네이버에서 보강된 값과 yfinance 원값이 카드에 표시될 때, 어떤 소스에서 온 값인지 툴팁 또는 보조 텍스트로 확인 가능.

**Acceptance Scenarios**:

1. **Given** PER가 네이버로 보강된 종목, **When** 사용자가 해당 카드에 hover / 탭, **Then** "출처: 네이버 파이낸스 · 기준일 YYYY-MM-DD"가 툴팁으로 표시.
2. **Given** ROE가 yfinance 값으로 표시된 종목, **When** 사용자가 카드에 hover, **Then** "출처: yfinance"가 툴팁으로 표시.
3. **Given** 네이버·yfinance 둘 다 있는 지표, **When** 값 충돌, **Then** 네이버 값을 우선 노출 (KR 종목에 한함).

---

### Edge Cases

- **신규 상장·거래정지 종목**: 네이버 종목 페이지가 404 또는 일부 지표 공란 → 각 지표 개별 결측 처리.
- **우선주(예: 005935)**: 네이버 URL 규칙이 보통주와 동일하나 일부 지표 의미가 다를 수 있음 — 보통주와 동일 파이프라인 적용, 값이 있으면 노출.
- **HTML 구조 변경**: 네이버가 페이지 구조를 바꾸면 파싱 실패 → 전체 종목 일괄 결측 발생 → 운영자 알림 또는 로그로 조기 감지 필요.
- **요청 과다**: 전체 스캔 동기 호출 시 네이버가 Rate Limit을 걸거나 IP 차단 가능 → 사용자 상세 진입 시점에만 on-demand 호출 + 캐시 적용.
- **KR 외 시장**: US 종목 등은 본 보강 대상이 아님 (기존 yfinance 유지).
- **ETF·암호화폐**: 가치 분석 자체 미지원이므로 네이버 조회 스킵.
- **캐시 만료 타이밍**: 가치 지표는 분기 단위 갱신 데이터이므로 과도한 실시간성 불필요.

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: KR(코스피·코스닥) 개별 상장주식에 한해, 가치 분석 탭에 필요한 **PER·PBR·EPS·BPS** 네 지표를 외부 한국어 금융 정보 소스에서 추가 수집하여 표시해야 한다.
- **FR-002**: 추가 수집 값이 존재하면 yfinance 원값보다 **우선 표시**한다. 추가 수집 값이 없는 지표는 yfinance 원값 또는 `—`로 폴백한다.
- **FR-003**: 추가 소스가 타임아웃·HTTP 오류·파싱 실패 등 어떤 이유로든 실패해도 **응답은 200**으로 유지되고, 다른 지표·기능에 영향을 주지 않는다 (Graceful Degradation).
- **FR-004**: 추가 수집은 **종목별 캐시**로 관리되며, 동일 종목의 연속 요청이 외부 소스에 중복 전송되지 않는다. 캐시 TTL 기본값은 **24시간**으로 한다 (분기 단위 갱신 데이터이므로 일 단위 신선도로 충분 + 외부 소스 부하 최소화).
- **FR-005**: 외부 소스 요청은 **사용자 상세 진입 시 on-demand**로 호출한다(전체 스캔 루프에서는 호출하지 않는다 — rate limit·부하 방지).
- **FR-006**: 각 지표 값에 대한 **데이터 출처(네이버/yfinance)**와 **수집·기준 시점**을 사용자에게 표시해야 한다. 표시 방식은 **카드별 출처 라벨 항상 노출** — 각 지표 카드의 기존 sublabel 뒤에 `·` 구분자로 소스명을 병합 표기(예: "주가/순이익 · 네이버", "자기자본이익률 · yfinance"). PC·모바일 동일 방식으로 카드 높이·스냅 정렬 변화 없음. 상세 기준일/수집시각은 카드 툴팁(PC 호버)에서 확인 가능하다.
- **FR-007**: 외부 소스 조회 실패는 **구조화된 로그**로 기록되어야 한다(어떤 종목·어떤 단계·무슨 에러).
- **FR-008**: 외부 소스의 페이지 구조 변경 등으로 **일정 비율 이상의 종목이 연속 실패**하면 관리자가 인지할 수 있도록 지표(실패율 카운터·최근 실패 리스트)가 관측 가능해야 한다.
- **FR-009**: ETF·암호화폐·지수·외환·US 종목에는 외부 소스 호출을 시도하지 않는다.
- **FR-010**: 가치 분석 탭의 기존 카드 레이아웃·툴팁·하이라이트 규칙은 그대로 유지되며, 본 기능으로 인한 UI 회귀는 없다.
- **FR-011**: 카드 표시 순서는 022에서 정한 **중요도 순**(시총 → PER → PBR → ROE → ROA → 영업이익률 → EPS → BPS → 배당 → 부채비율 → 섹터)을 변경 없이 그대로 유지한다.

### Key Entities

- **NaverFundamentalsPayload**: 특정 KR 종목의 네이버 보강 스냅샷 (자세한 필드는 data-model.md §1). 속성: 종목코드, PER, PBR, EPS, BPS, reporting_period(예: "2025-Q4"), fetched_at(UTC), source("naver").
- **MetricSourceMap**: 가치 분석 탭의 각 지표가 어느 데이터 소스에서 왔는지 기록 (data-model.md §2). 속성: `{metric_name: "naver" | "yfinance"}` 딕셔너리. 응답의 `metric_sources` 필드로 노출.

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: KR 대형주 샘플 30종목에서 PER/PBR/EPS/BPS **4개 지표 중 평균 3개 이상이 결측 없이** 채워진다 (현재 다수 종목에서 0개).
- **SC-002**: 가치 분석 탭 진입 시 외부 보강 포함 총 데이터 표시 시간이 **1.5초 이내(p95)** — 기존 성능 목표를 유지.
- **SC-003**: 외부 소스 **일시적 장애 상황에서도** 가치 분석 탭의 렌더 성공률 100% (다른 지표는 정상 표시, 보강 대상만 `—`).
- **SC-004**: 외부 소스 호출은 동일 종목에 대해 **캐시 TTL 내 1회 이하**로 제한된다(중복 호출률 0%).
- **SC-005**: 사용자가 카드 수치의 출처를 **2초 이내(hover/tap)**에 확인할 수 있다.
- **SC-006**: 외부 소스 호출이 전체 시장 스캔 경로에 포함되지 않아 스캔 완료 시간이 본 기능 도입 전 대비 **증가 없음** (편차 ±5% 이내).

## Assumptions

- 본 기능은 KR 개별 상장주식(코스피·코스닥)에 한정한다. US·Crypto·ETF·지수·외환은 범위 밖.
- 대상 외부 소스는 공개된 한국어 금융 정보 사이트 중 **사용자 측(클라이언트) 접근이 가능한 무료 공개 페이지**를 대상으로 한다(별도 계약·유료 API 미도입).
- 사용 및 수집은 **공개 정보의 화면 크롤링 수준**이며 상업적 대량 재판매는 본 범위 밖.
- 수집 결과의 **정확성 보장은 원 소스에 의존**하며, 본 서비스는 시점·출처를 함께 표기함으로써 사용자 판단을 돕는다.
- 네트워크 장애 처리의 기본 정책은 "보강 실패는 조용히 폴백, 원 데이터 기능은 그대로"이다.
- 외부 소스의 이용약관·robots 정책 준수는 개발자 책임으로 간주한다 (설계 시 준수 가능한 호출 패턴을 채택).

## Out of Scope (명시적 비포함)

- US·Crypto·지수·외환·ETF 종목에 대한 펀더멘털 보강.
- 재무제표 상세(손익계산서·현금흐름표) 및 다년도 추세 차트 노출.
- 외부 소스의 **실시간 웹소켓/피드** 도입.
- 애널리스트 컨센서스·목표주가·뉴스 통합.
- 사용자별 즐겨찾기·메모와의 연동.

## Dependencies

- 기존 `/company/{symbol}` 응답 스키마와 가치 분석 탭(022 기능)이 MVP 기반으로 동작 중이어야 함.
- 기존 yfinance 기반 metrics는 그대로 유지되며 본 기능은 그 위의 **보강 레이어**로 동작.

## Clarifications

### Session 2026-04-15

- **Q1: 네이버 파이낸스 보강 값 캐시 TTL 기본값** → **24시간**.
  - 분기 단위 갱신 데이터이므로 일 단위 신선도로 충분.
  - 네이버 호출 최소화로 rate limit·IP 차단 위험 완화.
  - FR-004 반영 완료.
- **Q2: 데이터 출처 표시 방식** → **카드별 출처 라벨 항상 노출**.
  - 각 지표 카드 sublabel 영역에 "네이버" 또는 "yfinance" 라벨 상시 표기.
  - 카드 툴팁에는 상세 기준일·수집시각 포함.
  - 가장 투명한 방식으로 사용자 판단 지원.
  - FR-006 반영 완료.
- **Q3: 가치 분석 탭 카드 정렬 방식** → **중요도 순 유지 (022 동일)**.
  - 순서 유지: ① 시총 → ② PER → ③ PBR → ④ ROE → ⑤ ROA → ⑥ 영업이익률 → ⑦ EPS → ⑧ BPS → ⑨ 배당 → ⑩ 부채비율 → ⑪ 섹터.
  - 네이버 보강은 "값 소스"만 대체 — 카드 순서·의미·우선순위는 022와 동일하게 유지해 사용자 레이아웃 학습성 보존.
  - 소스 차이는 FR-006의 카드별 출처 라벨로 구분.
- **Q4: 모바일에서 출처 라벨 표시 방식** → **sublabel 영역에 `·` 구분자로 병합**.
  - 기존 sublabel("주가/순이익" 등) 뒤에 `·` 구분자로 소스명을 붙여 표기 — 예: "주가/순이익 · 네이버", "자기자본이익률 · yfinance".
  - PC·모바일 동일 방식 → 카드 높이 변화 없음, 스냅 스크롤 정렬 재조정 불필요.
  - 구현 추가 비용 없음 (MetricCard.sublabel 문자열만 조합).
  - 기준일·수집시각은 카드 툴팁(PC 호버)에만 노출 — 모바일은 값 자체가 sublabel에 항상 보이므로 추가 탭 불필요.

# Feature Specification: BUY Signal Age Label on Stock Cards

**Feature Branch**: `031-signal-days-label`
**Created**: 2026-04-23
**Status**: Draft
**Input**: User description: "관심종목, 추천종목, 눌림목, 대형주에 있는 종목카드에 buy 신호가 뜬지 몇일이 지났는지 오늘기준으로 표시하는 라벨추가"

---

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 스캔 결과 카드에 BUY 신호 경과일 표시 (Priority: P1)

추천종목·눌림목·대형주 화면에서 각 종목 카드에 "BUY 신호가 오늘로부터 몇 일 전에 발생했는지"를 나타내는 라벨이 표시된다. 사용자는 종목 상세 화면에 들어가지 않고도 신호의 신선도를 즉시 파악할 수 있다.

**Why this priority**: 스캔 결과 종목에는 이미 신호 발생일 데이터가 존재한다. 백엔드 변경 없이 프론트엔드만 수정하면 되므로 즉시 적용 가능하며, 신호 신선도는 투자 판단에 직접 기여하는 핵심 정보다.

**Independent Test**: 추천종목 화면에서 종목 카드를 확인했을 때 "N일 전" 형식의 경과일 라벨이 표시되면 검증 완료.

**Acceptance Scenarios**:

1. **Given** 추천종목 카드에 BUY 신호 발생일이 오늘인 경우, **When** 사용자가 화면을 보면, **Then** "오늘" 라벨이 표시된다.
2. **Given** 추천종목 카드에 BUY 신호 발생일이 3일 전인 경우, **When** 사용자가 화면을 보면, **Then** "3일 전" 라벨이 표시된다.
3. **Given** 신호 발생일 데이터가 없는 카드의 경우, **When** 사용자가 화면을 보면, **Then** 경과일 라벨이 표시되지 않는다.
4. **Given** 눌림목·대형주 카드에도 동일한 규칙이 적용되어, **When** 사용자가 해당 섹션을 보면, **Then** 동일한 형식으로 경과일 라벨이 표시된다.
5. **Given** 경과일이 7일 이하인 카드와 8일 이상인 카드가 함께 보이는 경우, **When** 사용자가 화면을 보면, **Then** 두 그룹의 라벨 색상이 시각적으로 구분된다.

---

### User Story 2 - 관심종목 카드에 BUY 신호 경과일 표시 (Priority: P2)

관심종목 목록 카드에도 동일한 "N일 전" 라벨이 표시된다. 단, 현재 관심종목 데이터에는 신호 발생일이 포함되지 않으므로 백엔드에서 해당 필드를 응답에 추가해야 한다.

**Why this priority**: P1 완료 후 스캔 화면과 관심종목 간 경험이 불일치해진다. P2로 분리한 이유는 백엔드 수정이 필요하기 때문이며, P1 이후 자연스럽게 이어진다.

**Independent Test**: 관심종목 목록에서 BUY 신호 상태인 종목 카드에 "N일 전" 라벨이 표시되면 검증 완료.

**Acceptance Scenarios**:

1. **Given** 관심종목 카드의 신호 상태가 BUY이고 신호 발생일이 존재하는 경우, **When** 사용자가 관심종목 목록을 보면, **Then** "N일 전" 라벨이 표시된다.
2. **Given** 관심종목 카드의 신호 상태가 SELL 또는 NEUTRAL인 경우, **When** 사용자가 목록을 보면, **Then** 경과일 라벨이 표시되지 않는다.
3. **Given** 신호 발생일 데이터가 없는 경우, **When** 사용자가 목록을 보면, **Then** 경과일 라벨이 표시되지 않는다.

---

### Edge Cases

- 신호 발생일이 오늘인 경우: "0일 전" 대신 "오늘"로 표시
- 경과일이 미래 날짜인 경우(데이터 오류): 라벨 미표시
- 경과일이 매우 큰 경우(예: 90일 이상): 라벨 흐리게 표시하되 숨기지는 않음
- 주말·공휴일 포함 여부: 달력 일수(calendar days) 기준 계산 (거래일 환산 불필요)

---

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 추천종목·눌림목·대형주 화면의 각 종목 카드에 BUY 신호 발생일로부터 오늘까지의 경과 일수를 표시하는 라벨이 있어야 한다.
- **FR-002**: 경과일은 오늘 날짜 기준 달력 일수로 계산한다 (0일 = 오늘, 1일 이상 = "N일 전").
- **FR-003**: 경과일 표시 형식: 0일 → "오늘", 1일 이상 → "N일 전" (예: "3일 전").
- **FR-004**: 신호 발생일 데이터가 없거나 날짜가 오늘보다 미래인 경우 라벨을 표시하지 않는다.
- **FR-005**: 경과일이 7일 이하인 경우 강조 색상, 8일 이상인 경우 흐린 색상으로 표시하여 신호 신선도를 직관적으로 전달한다.
- **FR-006**: 경과일 라벨은 기존 신호 칩(BUY, SQZ BUY 등) 인근에 배치하여 신호와의 연관성을 시각적으로 명확히 한다.
- **FR-007**: 관심종목 카드에도 동일한 경과일 라벨을 표시한다. 단, 신호 상태가 BUY인 종목에만 표시한다.
- **FR-008**: 관심종목의 경과일 표시를 위해 백엔드 관심종목 응답에 신호 발생일 필드가 포함되어야 한다.

### Key Entities

- **신호 경과일(Signal Age)**: 신호 발생일(YYYY-MM-DD)과 오늘 날짜의 차이(정수, 달력 일수). 음수 또는 데이터 없음은 미표시.
- **종목 카드(Stock Card)**: 추천종목·눌림목·대형주의 BuyCard, 관심종목의 MiniWatchCard. 신호 경과일 라벨을 포함하는 UI 단위.

---

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 추천종목·눌림목·대형주 화면의 모든 종목 카드 중 신호 발생일 데이터가 있는 카드에서 경과일 라벨이 100% 표시된다.
- **SC-002**: 관심종목 화면에서 BUY 신호 상태 종목에 경과일 라벨이 표시된다.
- **SC-003**: 경과일 7일 이하(신선)와 8일 이상(오래됨) 종목이 색상으로 구분되어 사용자가 신호 신선도를 한눈에 파악할 수 있다.
- **SC-004**: 신호 발생일 없는 카드에서 라벨 관련 빈 값이나 오류 표시가 0건이다.

---

## Assumptions

- 경과일은 달력 일수로 계산한다. 거래일 환산은 복잡도 대비 UX 개선이 적다고 판단.
- 7일 이하를 "신선한 신호"로 정의 (약 1거래주 이내). 이 기준값은 추후 조정 가능.
- 관심종목 신호 발생일 추가는 기존 `/signals` 응답에 필드 1개(`last_signal_date`)를 추가하는 방식.
- 라벨은 항상 표시되며 별도 사용자 설정은 없다.

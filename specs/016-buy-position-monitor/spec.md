# Feature Specification: BUY 포지션 상태 모니터링

**Feature Branch**: `016-buy-position-monitor`
**Created**: 2026-04-06
**Status**: Draft
**Input**: BUY 신호 이후 SELL 신호가 나오지 않은 종목들을 스캔하고 관심종목으로 추적하고 있는데 BUY 신호 이후에 현재 상태가 어떤 상태인지 객관적으로 알수 있는 기능 BUY 신호 당시에 긍정적인 요소가 사라진건 없는지 확인할수 있어야함

## Overview

관심종목 중 BUY 신호 이후 아직 SELL 신호가 나오지 않은 종목들의 현재 상태를 객관적으로 진단하는 기능이다. BUY 신호가 발생했던 당시의 긍정적 지표 요소들이 현재도 유효한지, 아니면 약화·소멸됐는지를 요소별로 비교하여 사용자가 포지션 유지 여부를 판단할 수 있도록 돕는다.

---

## User Scenarios & Testing

### User Story 1 — BUY 포지션 현황 목록 조회 (Priority: P1)

관심종목 중 BUY 신호 이후 SELL이 없는 종목들을 한 화면에서 확인한다. 각 종목의 BUY 신호 발생일, 현재 수익률(BUY 당시가 → 현재가), 전반적인 상태(건강·주의·위험)를 한눈에 파악할 수 있다.

**Why this priority**: 포지션을 보유 중인 사용자가 가장 먼저 보고 싶은 정보. 별도 화면 없이 이것만 있어도 즉각적인 판단이 가능하다.

**Independent Test**: 관심종목에 BUY 신호 종목이 1개 이상 있을 때, 목록이 표시되고 각 항목에 BUY 신호 날짜·현재 수익률·상태 배지가 나타나면 합격.

**Acceptance Scenarios**:

1. **Given** 관심종목 A가 2026-04-02에 BUY 신호를 받고 이후 SELL 신호가 없는 상태일 때, **When** 모니터링 화면을 열면, **Then** A가 목록에 표시되고 BUY 일자(04-02)와 BUY 당시가 대비 현재 수익률이 표시된다.
2. **Given** 관심종목 B가 SELL 신호를 받은 상태일 때, **When** 모니터링 화면을 열면, **Then** B는 목록에 표시되지 않는다.
3. **Given** BUY 포지션 종목이 없을 때, **When** 모니터링 화면을 열면, **Then** "현재 추적 중인 BUY 포지션이 없습니다" 안내 문구가 표시된다.

---

### User Story 2 — 개별 종목 지표 요소 상태 체크 (Priority: P2)

BUY 신호 당시 긍정적이었던 각 지표 요소(RSI, BB %B, 추세, 스퀴즈, MACD 방향, 거래량)의 현재 값을 BUY 당시 값과 나란히 보여준다. 각 요소가 아직 긍정적인지(유지), 약화됐는지(주의), 소멸됐는지(이탈)를 색상과 아이콘으로 명시한다.

**Why this priority**: 수익률만 보면 언제 팔아야 하는지 판단 근거가 없다. 지표 요소별 비교가 있어야 근거 있는 의사결정이 가능하다.

**Independent Test**: 종목 카드 또는 상세 화면에서 BUY 당시 값 / 현재 값 / 상태(유지·주의·이탈) 가 6개 지표 모두에 대해 표시되면 합격.

**Acceptance Scenarios**:

1. **Given** BUY 신호 당시 RSI가 38이었던 종목이 현재 RSI 68일 때, **When** 해당 종목의 요소 상태를 보면, **Then** RSI 항목이 "주의" 또는 "이탈"로 표시된다.
2. **Given** BUY 신호 당시 EMA 정배열이었던 종목이 현재도 정배열일 때, **When** 요소 상태를 보면, **Then** 추세 항목이 "유지"로 표시된다.
3. **Given** BUY 신호 당시 스퀴즈 레벨이 2(MID)였던 종목이 현재 스퀴즈 해소(레벨 0)된 경우, **When** 요소 상태를 보면, **Then** 스퀴즈 항목이 "해소"로 표시된다 (해소는 BUY 이후 자연스러운 변화이므로 중립 처리).

---

### User Story 3 — 요소 이탈 시 텔레그램 경보 (Priority: P3)

BUY 포지션 유지 중인 종목에서 핵심 긍정 요소가 소멸되면 텔레그램으로 사전 경보를 발송한다. SELL 신호가 공식 발생하기 전에 사용자가 리스크를 인지할 수 있도록 한다.

**Why this priority**: P1·P2 없이는 동작할 수 없으며, 화면을 자주 확인하지 않는 사용자를 위한 보조 수단이다.

**Independent Test**: BUY 포지션 종목에서 데드크로스(EMA 역배열) 전환이 발생하면 10분 이내에 텔레그램 경보 메시지가 수신되면 합격.

**Acceptance Scenarios**:

1. **Given** 종목 A가 BUY 포지션 상태이고 EMA 정배열이었는데, **When** EMA 역배열(데드크로스)로 전환되면, **Then** "⚠️ [종목명] BUY 포지션 경보 — 추세 이탈" 텔레그램 메시지가 발송된다.
2. **Given** 종목 B의 RSI가 BUY 당시 40 이하였는데, **When** 현재 RSI가 70 이상으로 진입하면, **Then** "⚠️ [종목명] BUY 포지션 경보 — RSI 과열" 메시지가 발송된다.
3. **Given** 같은 종목에서 동일 경보가 이미 발송된 경우, **When** 동일 조건이 지속되면, **Then** 24시간 이내 중복 발송되지 않는다.

---

### Edge Cases

- BUY 신호 당시 가격 데이터가 없는 경우 수익률을 "-"로 표시하고, 요소 비교는 현재 값만 표시한다.
- 관심종목에서 삭제된 종목은 BUY 포지션 목록에서도 자동 제거된다.
- 장 마감 후 및 주말에는 최근 마지막 거래일 데이터 기준으로 표시하며 "마지막 업데이트 시각"을 명시한다.
- 동일 종목에서 BUY 신호가 여러 번 발생한 경우 가장 최근 BUY 신호를 기준으로 한다.
- 암호화폐는 24시간 거래이므로 장 마감 개념 없이 실시간 데이터 기준으로 표시한다.
- 기존 BUY 신호 이력에 지표 스냅샷이 없는 종목은 BUY 당시 값을 "-"로 표시하고, 현재 값만으로 상태를 표시한다.

## Requirements

### Functional Requirements

- **FR-001**: 시스템은 관심종목 중 신호 이력에서 가장 최근 신호가 BUY이고 이후 SELL이 없는 종목을 자동으로 식별해야 한다.
- **FR-002**: 각 BUY 포지션 종목에 대해 BUY 신호 발생일, BUY 당시 종가, 현재 종가, 수익률(%)을 표시해야 한다.
- **FR-003**: BUY 신호 발생 시점의 지표 값(RSI, BB %B, EMA 추세, 스퀴즈 레벨, MACD 히스토그램 방향, 거래량 비율) 6개를 기록하고 현재 값과 비교해야 한다.
- **FR-004**: 각 지표 요소의 상태를 유지 / 주의 / 이탈 3단계로 분류해야 한다.
- **FR-005**: 전체 요소 상태를 종합한 포지션 건강 점수(0~100)를 산출하여 표시해야 한다. 80점 이상 = 건강, 50~79점 = 주의, 50점 미만 = 위험.
- **FR-006**: 포지션 건강 점수가 이전 스캔 대비 20점 이상 하락하거나, 핵심 요소(EMA 추세, RSI)가 이탈 상태로 전환되면 텔레그램 경보를 발송해야 한다.
- **FR-007**: 경보는 종목당 24시간 이내 동일 조건 중복 발송을 방지해야 한다.
- **FR-008**: BUY 포지션 목록은 포지션 건강 점수 오름차순(위험 종목 상단)으로 기본 정렬되어야 한다.
- **FR-009**: 사용자는 개별 종목의 요소 상세 비교 화면(BUY 당시 vs 현재)에 접근할 수 있어야 한다.
- **FR-010**: 새로운 BUY 신호 발생 시 해당 시점의 지표 값이 자동으로 스냅샷 저장되어야 한다.

### Key Entities

- **BUY 포지션 (BuyPosition)**: 관심종목 중 마지막 신호가 BUY이고 SELL이 없는 추적 대상. BUY 일자, BUY 당시 가격, BUY 당시 지표 스냅샷을 포함한다.
- **지표 스냅샷 (IndicatorSnapshot)**: BUY 신호 발생 시점에 기록된 RSI, BB %B, EMA 추세, 스퀴즈 레벨, MACD 방향, 거래량 비율의 값 묶음.
- **요소 상태 (FactorStatus)**: 현재 지표 값이 BUY 당시 대비 긍정 조건을 얼마나 유지하는지를 나타내는 분류 (유지 / 주의 / 이탈).
- **포지션 건강 점수 (HealthScore)**: 각 요소 상태를 가중 합산한 0~100 점수. 위험 정도를 직관적으로 전달한다.

## Success Criteria

### Measurable Outcomes

- **SC-001**: BUY 포지션 목록 화면이 3초 이내에 로드되어 관심종목의 포지션 상태를 즉시 확인할 수 있다.
- **SC-002**: 요소 이탈 경보는 이탈 발생 후 10분 이내(다음 스캔 사이클 내)에 텔레그램으로 발송된다.
- **SC-003**: 6개 지표 요소 모두에 대해 BUY 당시 값 / 현재 값 / 상태가 누락 없이 표시된다.
- **SC-004**: 동일 종목 동일 조건에 대한 중복 경보가 24시간 이내 발송되지 않는다.
- **SC-005**: BUY 포지션이 없을 때 명확한 안내 메시지가 표시되어 사용자가 혼란을 느끼지 않는다.

---

## Assumptions

- BUY 신호 당시 지표 값은 신호 발생 시점에 별도 스냅샷으로 저장해야 하며, 기존 `signal_history` 테이블에 지표 값이 없다면 신규 저장 로직이 필요하다.
- 현재 지표 값은 기존 10분 스캔 데이터를 재사용한다 (별도 실시간 요청 없음).
- 포지션 건강 점수 기본 가중치: EMA 추세(30점), RSI(20점), BB %B(15점), MACD(15점), 스퀴즈(10점), 거래량(10점).
- 요소 이탈 판단 기준(기본값):
  - RSI: BUY 당시 < 50이었는데 현재 >= 65 → 주의 / >= 70 → 이탈
  - EMA: 정배열 → 역배열 전환 → 이탈
  - BB %B: BUY 당시 < 0.4였는데 현재 >= 0.8 → 주의
  - MACD 히스토그램: 양 → 음 전환 → 주의
  - 거래량: 평균 대비 0.5배 미만 3회 연속 → 주의
- 암호화폐는 동일 기준 적용하되 RSI 이탈 기준을 75로 완화한다 (높은 변동성 반영).
- 모니터링 갱신 주기는 기존 10분 스캔과 동일하게 유지한다.
- 이 기능은 투자 권유가 아닌 참고 정보 제공 목적이며, 모든 화면에 해당 문구를 표시한다.

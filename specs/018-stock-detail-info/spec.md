# Feature Specification: 종목 상세 — 투자 지표 및 회사 정보 패널

**Feature Branch**: `018-stock-detail-info`  
**Created**: 2026-04-07  
**Status**: Draft  
**Input**: 토스증권 종목상세에 투자 지표, 회사정보를 동일하게 보여주는 기능

## Background

현재 종목 상세 페이지(SignalDetail)에는 PER·PBR·EPS·BPS 4개 투자지표와 52주 범위, 가격제한 정보만 표시된다. 한국투자증권(KIS) API가 미설정인 경우 투자지표 전체가 표시되지 않으며, 회사 소개·업종·배당·ROE 등 투자 판단에 필요한 정보가 없다.

토스증권의 "종목정보" 탭처럼 회사 개요, 풍부한 투자 지표, 매출 구성을 누구나(KIS 미설정 포함) 확인할 수 있도록 한다.

### Session 2026-04-08

## User Scenarios & Testing *(mandatory)*

### User Story 1 — 종목 상세에서 회사 정보 확인 (Priority: P1)

사용자가 종목 상세 페이지를 열면 별도 탭 이동 없이 **회사 소개·업종·섹터** 정보를 바로 확인할 수 있다.

**Why this priority**: 투자 판단의 첫 단계인 "어떤 회사인가"를 빠르게 파악하는 것이 핵심 니즈다. 모든 종목(KR·US)에서 동작해야 하며, KIS 설정 여부와 무관하게 작동해야 한다.

**Independent Test**: 종목 상세 페이지에서 회사 설명 텍스트, 업종·섹터 배지, 국가·거래소 정보가 표시되는지 단독으로 검증 가능.

**Acceptance Scenarios**:

1. **Given** 한국 주식 종목 상세 페이지를 열었을 때, **When** 스크롤하면, **Then** 회사 로고(또는 텍스트 아바타)·회사명·업종·사업 개요 텍스트·섹터·국가/거래소 정보가 표시된다.
2. **Given** 미국 주식 종목 상세 페이지를 열었을 때, **When** 스크롤하면, **Then** 회사 로고(또는 텍스트 아바타)·회사명·업종·사업 개요(영문)·섹터·직원 수·웹사이트 링크가 표시된다.
3. **Given** 회사 정보를 불러오는 중일 때, **When** 데이터가 로딩 중이면, **Then** 스켈레톤 또는 로딩 표시가 나타난다.
4. **Given** 암호화폐 종목을 열었을 때, **When** 스크롤하면, **Then** 회사 정보 섹션이 표시되지 않는다(암호화폐는 기업 없음).
5. **Given** 모바일에서 종목 상세를 열었을 때, **When** 실적 차트 아래로 스크롤하면, **Then** "회사 정보" 아코디언 헤더가 접힌 상태로 표시된다.
6. **Given** 모바일에서 아코디언 헤더를 탭했을 때, **When** 헤더를 누르면, **Then** 회사 정보 내용이 펼쳐진다.

### User Story 2 — 확장된 투자 지표 확인 (Priority: P1)

사용자가 PER·PBR 외에 ROE·배당수익률·영업이익률·부채비율 등 핵심 투자 지표를 종목 상세에서 한눈에 확인할 수 있다.

**Why this priority**: 기존 4개 지표(PER·PBR·EPS·BPS)는 KIS 설정 시에만 표시되어 미국 주식이나 KIS 미설정 환경에서 아무것도 보이지 않는다. 모든 환경에서 동작하는 풍부한 지표 세트가 필요하다.

**Independent Test**: KIS 미설정 환경에서 미국 주식 상세 페이지를 열어 투자 지표 카드들이 표시되는지 검증 가능.

**Acceptance Scenarios**:

1. **Given** KIS가 설정되지 않은 환경에서, **When** 미국 주식 상세를 열면, **Then** PER·PBR·ROE·배당수익률·시가총액이 표시된다.
2. **Given** 투자 지표 값이 존재할 때, **When** PER이 10 미만이면, **Then** 해당 지표에 "저평가" 강조 표시가 나타난다.
3. **Given** 투자 지표 값이 존재할 때, **When** PER이 30 초과이면, **Then** 해당 지표에 "고평가" 강조 표시가 나타난다.
4. **Given** 특정 지표 데이터가 없을 때, **When** 화면에 표시되면, **Then** "-"로 표시되며 레이아웃이 깨지지 않는다.
5. **Given** 한국 주식을 열었을 때, **When** 투자 지표를 확인하면, **Then** 금액 단위가 원(KRW)으로 표시된다.

### User Story 3 — 매출 구성 차트 확인 (Priority: P2)

사용자가 해당 기업의 사업 세그먼트별 매출 비중을 도넛 차트로 시각적으로 확인할 수 있다.

**Why this priority**: 토스증권의 차별화 기능 중 하나로, 기업의 주력 사업과 매출 다각화 여부를 직관적으로 파악하는 데 도움을 준다. P1이 완성된 후 추가적 가치를 제공한다.

**Independent Test**: 매출 구성 데이터가 있는 미국 주식에서 도넛 차트와 범례가 정상 표시되는지 단독 검증 가능.

**Acceptance Scenarios**:

1. **Given** 세그먼트별 매출 데이터가 있는 종목을 열었을 때, **When** 스크롤하면, **Then** 매출 구성 도넛 차트와 각 세그먼트 이름·비중(%)이 표시된다.
2. **Given** 매출 구성 데이터가 없는 종목을 열었을 때, **When** 스크롤하면, **Then** 매출 구성 섹션이 표시되지 않는다.
3. **Given** 차트가 표시될 때, **When** 데이터 기준일(출처 포함)을 확인하면, **Then** "YYYY.MM 기준" 날짜 정보가 함께 표시된다.

---

### Edge Cases

- 회사 정보 API 응답 실패 시 해당 섹션만 숨김 처리 (나머지 페이지는 정상 표시).
- 로고 이미지 URL이 존재하지만 로드에 실패할 경우(네트워크 오류, 404 등) 텍스트 아바타로 자동 대체하며, 깨진 이미지 아이콘이 노출되지 않아야 한다.
- 사업 개요 텍스트가 매우 길 경우 일정 줄 이후 "더 보기" 접기/펼치기 처리.
- KIS API와 yfinance 데이터가 동시에 있을 경우, 기존 KIS 데이터 우선 유지 (중복 표시 금지).
- 암호화폐 종목: 회사 정보·매출 구성 섹션 비표시.
- 배당을 지급하지 않는 종목: 배당수익률 "-" 표시 (0% 아님).
- 한국 주식의 영문 사업 개요만 있을 경우: 영문으로 표시하되, 없을 경우 섹션 숨김.

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 시스템은 종목 상세 페이지에서 KIS 설정 여부와 무관하게 투자 지표를 표시해야 한다.
- **FR-002**: 투자 지표 섹션은 최소 PER·PBR·ROE·배당수익률·시가총액·EPS·BPS를 포함해야 한다. PER·EPS는 TTM(최근 12개월 실적) 기준으로 산출하며, 지표 레이블에 "TTM" 또는 "최근 12개월" 기준임을 명시해야 한다.
- **FR-003**: 시스템은 회사 정보 섹션(로고/아바타·사업 개요·업종·섹터·국가)을 종목 상세 페이지에 표시해야 한다.
- **FR-013**: 회사 로고는 yfinance 제공 URL에서 우선 로드하며, 로드 실패·미제공 시 회사명 첫 글자와 고정 색상으로 구성된 원형 텍스트 아바타를 표시해야 한다.
- **FR-004**: 회사 정보와 투자 지표 데이터는 한국(KR)·미국(US) 주식 모두에 대해 제공되어야 한다.
- **FR-005**: 매출 세그먼트 데이터가 있는 종목에서 매출 구성 도넛 차트를 표시해야 한다.
- **FR-006**: 각 투자 지표 값에는 단위(배, %, 원, $)가 함께 표시되어야 한다.
- **FR-007**: 저평가·고평가 기준에 해당하는 지표(PER 10 미만 = 저평가, PER 30 초과 = 고평가 등)에는 시각적 강조 표시가 되어야 한다.
- **FR-008**: 사업 개요 텍스트가 4줄을 초과할 경우 접기/펼치기 기능을 제공해야 한다.
- **FR-009**: 미국 주식 회사 정보에는 직원 수·웹사이트 링크가 포함되어야 한다.
- **FR-010**: 암호화폐 종목에는 회사 정보·매출 구성 섹션을 표시하지 않아야 한다.
- **FR-011**: 투자 지표 데이터는 1시간 이상 캐싱되어 반복 요청 시 즉시 표시되어야 한다.
- **FR-012**: 데이터 로딩 중에는 스켈레톤 UI를 표시하고, 실패 시 해당 섹션만 조용히 숨겨야 한다.
- **FR-014**: PC에서 신규 섹션(회사 정보·확장 투자 지표·매출 구성)은 실적 차트(FinancialChart) 아래에 배치되어야 한다.
- **FR-015**: 모바일에서 신규 섹션은 실적 차트 아래에 위치하며, 기본 접힘(collapsed) 상태의 아코디언으로 표시되어야 한다. 사용자가 헤더를 탭하면 펼쳐진다.

### Key Entities

- **CompanyInfo**: 회사명, 로고 URL(없으면 null), 사업 개요, 업종(industry), 섹터(sector), 국가, 거래소, 직원 수, 웹사이트 URL
- **InvestmentMetrics**: PER, PBR, ROE, ROA, EPS, BPS, 배당수익률, 시가총액, 영업이익률, 부채비율
- **RevenueSegment**: 세그먼트명, 매출 금액, 전체 대비 비중(%), 기준 날짜

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 종목 상세 페이지에서 회사 정보와 투자 지표가 3초 이내에 표시된다.
- **SC-002**: KIS 설정 없이도 한국·미국 주식 모두에서 투자 지표 7개 이상이 표시된다.
- **SC-003**: 기존 KIS 기반 투자 지표 표시가 깨지지 않고 그대로 유지된다 (회귀 없음).
- **SC-004**: 재방문(캐시) 시 회사 정보와 투자 지표가 0.5초 이내에 표시된다.
- **SC-005**: 데이터가 없는 지표에서 레이아웃 깨짐 없이 "-"가 표시된다.
- **SC-006**: 모바일(375px)과 PC(1280px) 화면 양쪽에서 투자 지표 카드가 정상 렌더링된다.

## Assumptions

- 회사 정보 및 투자 지표의 주 데이터 소스로 yfinance를 활용한다 (무료, 이미 설치됨).
- 한국 주식의 경우 yfinance 데이터 품질이 낮을 수 있어 일부 지표(ROE, 배당 등)는 데이터가 없을 수 있다 — "-"로 표시하는 것을 허용.
- 매출 구성 차트는 yfinance의 세그먼트 데이터가 있는 종목(주로 미국 대형주)에만 표시된다.
- 섹터 내 시가총액 순위(토스증권 UI의 순위 카드)는 이번 범위에서 제외한다 — 전체 섹터 종목 일괄 조회가 필요한 고비용 기능.
- 기존 KIS 기반 `StockFundamentals` 컴포넌트(차트 직후)와 `FinancialChart`(실적 차트)는 그대로 유지하고, 새 섹션은 `FinancialChart` 아래에 추가한다.
- 데이터 캐시 유효기간은 1시간으로 설정한다 (yfinance 기업정보는 하루 단위 변동).

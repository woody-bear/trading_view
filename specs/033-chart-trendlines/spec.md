## Clarifications

### Session 2026-04-25

- Q: 차트에 그릴 추세선 구조는 3선(독립)인가 4선(2채널)인가? → A: 4선 — 하락채널(하락추세선+평행선) + 상승채널(상승추세선+평행선). 이미지 원본과 동일한 구조.
- Q: 5단계 "상승추세선 돌파"는 어느 선 기준인가? → A: 상승채널 **상단 평행선** 돌파 — 채널 상단을 뚫고 강하게 상승하는 것이 매수급소 완성 시점.
- Q: 추세 단계 패널의 종목 상세 화면 내 위치는? → A: 차트 **바로 아래 스크롤 섹션** — 차트와 같은 탭, 스크롤하면 패널 등장.
- Q: 추세선 기간 구분 방식은? → A: **기간 선택 탭 (1·3·6·12개월)** — 최초 진입 시 12개월치 데이터를 한 번에 수신, 탭 전환은 클라이언트 사이드에서 차트 표시 범위만 변경(네트워크 재요청 없음). 추세선은 선택 기간의 스윙 포인트로 재계산.
- Q: 추세 변곡점(단계 전환 시점)에서의 거래량 표시 방식은? → A: **차트 볼륨 바 강조 + 패널 수치 동시 표시** — 단계 전환 봉의 볼륨 바를 특별 색상으로 하이라이트하고, 패널 해당 단계에 평균 대비 배율(예: "거래량 2.3배") 수치를 함께 표시.

## User Scenarios & Testing *(mandatory)*

### User Story 1 — 차트에 자동 추세선 표시 (Priority: P1)

투자자가 종목 상세 화면의 일봉 차트를 열었을 때, **1·3·6·12개월 기간 탭**을 선택하면 해당 기간의 차트 범위와 함께 최근 고점·저점을 잇는 **하락채널(하락추세선+평행선)** 과 **상승채널(상승추세선+평행선)** 이 자동으로 그려져 추세의 방향과 강도를 한눈에 파악할 수 있다. 탭 전환 시 재로딩 없이 즉시 차트 범위와 추세선이 변경된다.

**Why this priority**: 추세선은 추세 단계 판단의 전제 조건이며, 차트에서 직접 시각적으로 확인하는 것이 투자자가 신호를 신뢰하는 핵심 근거가 된다.

**Independent Test**: 종목 상세 화면 차트에서 1·3·6·12개월 탭을 전환할 때 네트워크 요청 없이 차트 범위와 하락채널(2선)·상승채널(2선)이 즉시 바뀌면 검증 완료. 추세선이 없는 경우(데이터 부족)에는 차트만 표시.

**Acceptance Scenarios**:

1. **Given** 60일 이상 거래 이력이 있는 종목 상세 화면을 열었을 때, **When** 기본 탭(3개월)으로 차트를 확인하면, **Then** 하락추세선(빨간색)과 그 평행선으로 구성된 하락채널, 상승추세선(초록색)과 그 평행선으로 구성된 상승채널이 차트 위에 오버레이 표시된다. 총 4선(하락채널 2선 + 상승채널 2선).
2. **Given** 차트가 로드된 상태에서, **When** 기간 탭을 1개월 → 6개월로 전환하면, **Then** 네트워크 요청 없이 차트 좌우 범위가 즉시 변경되고 해당 기간의 추세선이 새로 그려진다.
3. **Given** 명확한 상승 구간이 없는 종목일 때, **When** 차트를 확인하면, **Then** 계산 가능한 채널만 표시되고 나머지는 생략된다.
4. **Given** 거래 이력이 60일 미만인 신규 종목일 때, **When** 차트를 열면, **Then** 추세선 없이 기존 차트만 표시되고 추세 단계 패널에 "분석 불가 — 데이터 부족" 메시지가 나온다.

### User Story 2 — 현재 추세 단계 패널 표시 (Priority: P1)

종목 상세 화면의 **차트 바로 아래 스크롤 섹션**에서 **현재 가격이 5단계 추세 전환 경로 중 어느 단계에 있는지** 를 명확히 표시한다.

5단계 순서:
1. 하락추세선 돌파 — 하락채널 상단선(하락추세선)을 종가가 상향 통과
2. 평행추세선 지지 — 하락채널 하단 평행선 ±2% 이내에서 반등 확인
3. 평행추세선 돌파 — 하락채널 하단 평행선을 종가가 상향 통과
4. 상승추세선 지지 — 상승채널 하단선(상승추세선) ±2% 이내에서 반등 확인
5. 상승추세선 돌파 — 상승채널 **상단 평행선**을 종가가 상향 통과 (매수급소 완성)

**Why this priority**: 투자자가 추세 반전의 진행 상황을 레이블과 진행 표시로 즉시 파악해 매수 타이밍 결정에 직접 활용할 수 있다.

**Independent Test**: 추세 단계 패널이 차트 분석 탭에 표시되고 현재 단계가 5단계 중 하나로 강조 표시되면 검증 완료.

**Acceptance Scenarios**:

1. **Given** 하락추세선을 최근에 돌파한 종목의 상세 화면에서, **When** 추세 단계 패널을 확인하면, **Then** "1단계: 하락추세선 돌파 ✓"가 완료로 표시되고, 돌파 시점 거래량 배율(예: "거래량 2.1×")이 함께 표시되며, 차트에서 해당 봉의 볼륨 바가 강조된다.
2. **Given** 평행추세선 위에서 지지를 받고 있는 종목일 때, **When** 패널을 확인하면, **Then** "2단계: 평행추세선 지지 진행 중"이 현재 단계로 강조된다.
3. **Given** 5단계를 모두 완료한 종목일 때, **When** 패널을 확인하면, **Then** 전체 5단계가 완료 표시되고 "추세 전환 완성" 요약이 표시된다.
4. **Given** 아직 하락추세선을 돌파하지 못한 종목일 때, **When** 패널을 확인하면, **Then** 0/5 단계로 표시되고 "하락 추세 진행 중"과 함께 다음 목표 가격 수준이 표시된다.

### User Story 3 — 각 추세선의 현재 기준 가격 표시 (Priority: P2)

각 추세선이 **현재 날짜 기준으로 어느 가격 수준에 있는지** 를 패널에 수치로 표시하여, 투자자가 어느 가격까지 도달해야 다음 단계로 진입하는지 구체적으로 알 수 있다.

**Why this priority**: 추세선 이름만으로는 행동 기준이 불분명하다. 가격 수준 병기 시 매수·손절 기준으로 직접 활용 가능하다.

**Independent Test**: 추세 단계 패널의 각 항목 옆에 현재 시점의 추세선 가격이 숫자로 표시되면 검증 완료.

**Acceptance Scenarios**:

1. **Given** 상세 화면에서 추세 단계 패널을 볼 때, **When** 각 단계 항목을 확인하면, **Then** 해당 추세선의 현재 가격 수준(예: "하락추세선 1,420,000원")이 함께 표시된다.
2. **Given** 이미 돌파 완료된 단계의 항목에서, **When** 가격을 확인하면, **Then** 돌파 시점의 가격이 기록으로 표시된다.

---

### Edge Cases

- 거래 이력이 60일 미만인 신규 상장 종목: 추세선 그리지 않음, 패널에 "데이터 부족" 표시
- 유효한 스윙 고점·저점이 2개 미만인 경우: 해당 채널 생략, 나머지만 표시
- 수평 횡보 구간에서 상승/하락 채널 구분이 불가능한 경우: 확인된 채널만 표시
- 장 중 가격 변동으로 단계 판정이 변경될 때: 차트 데이터 갱신 주기에 맞춰 패널 자동 갱신
- 1개월 탭에서 유효 스윙 포인트가 2개 미만인 경우: 해당 채널 생략, 패널에 "기간을 늘려 확인하세요" 안내 표시
- 모바일 화면: 추세선은 차트에 표시, 단계 패널은 스크롤 가능한 카드 형태로 표시

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 종목 상세 화면 최초 진입 시 12개월치(약 252 거래일) 일봉 데이터를 한 번에 수신해야 한다. 이후 1·3·6·12개월 탭 전환은 추가 네트워크 요청 없이 클라이언트에서 처리해야 한다.
- **FR-002**: 기간 선택 탭(1·3·6·12개월, 기본값 3개월)이 차트 상단에 표시되어야 한다.
- **FR-003**: 기간 탭 전환 시 차트 표시 범위(시작일~종료일)가 즉시 변경되고, 해당 기간의 스윙 포인트 기반 하락채널(2선)·상승채널(2선) 총 4선이 재계산되어 오버레이 표시되어야 한다.
- **FR-004**: 현재 종가와 각 추세선 위치를 비교하여 5단계 추세 전환 중 현재 단계를 자동 판정해야 한다.
- **FR-005**: 추세 단계 판정 결과는 차트 바로 아래 스크롤 섹션에 단계별 진행 상태로 표시되어야 한다.
- **FR-006**: 각 추세선의 현재 시점 연장 가격(원화 또는 달러)이 패널에 숫자로 표시되어야 한다.
- **FR-007**: 데이터 부족 또는 추세선 계산 불가 시 패널에 사유와 함께 안내 메시지를 표시해야 한다.
- **FR-008**: 추세선과 패널은 PC와 모바일 양쪽에서 정상 표시되어야 한다.
- **FR-009**: 추세선·단계 데이터는 차트 데이터와 동일 주기로 갱신되어야 한다.
- **FR-010**: 단계 전환이 발생한 봉(변곡점 캔들)의 볼륨 바는 차트에서 특별 색상으로 하이라이트되어야 한다.
- **FR-011**: 추세 단계 패널의 각 완료 단계에는 해당 시점의 거래량이 직전 5거래일 평균 대비 배율(예: "거래량 2.3×")로 표시되어야 한다.

### Key Entities

- **추세선 채널 (TrendChannel)**: 종류(상승채널/하락채널), 주선(메인 추세선) 좌표 2개(날짜·가격), 평행선 좌표 2개(날짜·가격), 현재 날짜 기준 각 선의 연장 가격, 유효 여부
- **추세 단계 (TrendPhase)**: 종목 심볼, 현재 단계 번호(0~5), 각 단계 완료 여부 및 완료 시점 가격, 완료 시점 거래량 배율(직전 5거래일 평균 대비), 판정 기준일
- **스윙 포인트 (SwingPoint)**: 날짜, 가격, 종류(고점/저점), 감지 기준(좌우 n봉 내 극값)

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 60일 이상 데이터가 있는 종목에서 추세선 4선이 차트 로드 완료 후 3초 이내에 표시된다. 기간 탭 전환 시 추세선 재계산은 즉시(1초 이내) 완료된다.
- **SC-002**: 추세 단계 패널이 현재 단계를 5단계 중 하나로 즉시 식별하여 표시하고, 완료된 각 단계에 거래량 배율이 함께 표시된다.
- **SC-006**: 단계 전환 봉의 볼륨 바가 차트에서 일반 봉과 시각적으로 구분 가능하다(육안 검증).
- **SC-003**: 패널에 표시된 추세선 기준 가격이 차트상 시각적 위치와 일치한다(육안 검증 가능).
- **SC-004**: 모바일과 PC 양쪽에서 레이아웃 깨짐 없이 패널과 차트 추세선이 표시된다.
- **SC-005**: 데이터 부족 종목에서 오류 메시지 없이 안내 텍스트가 표시된다.

## Assumptions

- 추세선은 사용자가 수동으로 그리는 것이 아니라 시스템이 자동 계산하여 표시한다.
- 기준 타임프레임은 일봉(1D) — 기존 차트와 동일.
- 최초 데이터 수신: 12개월치(약 252 거래일). 현재 차트 API(fetchers)는 2년치(period="2y")를 제공하므로 별도 기간 연장 불필요. 차트 API limit 파라미터를 300 → 260으로 조정하면 충분.
- 탭 전환 최적화: 12개월 전체 캔들 배열을 메모리에 보유 후 선택 기간만 slice하여 차트에 전달 (재요청 없음).
- 스윙 포인트 감지 기준: 좌우 최소 5봉 이내 최고점/최저점 (선택된 기간 내 데이터만 대상).
- 가장 최근 유효 고점 2개를 이어 하락추세선, 그 평행선은 인근 저점으로 평행 이동하여 하락채널 구성.
- 가장 최근 유효 저점 2개를 이어 상승추세선, 그 평행선은 인근 고점으로 평행 이동하여 상승채널 구성.
- 하락채널 색상: 빨간색 계열 / 상승채널 색상: 초록색 계열 (프로젝트 기존 상승=초록, 하락=빨강 관례).
- "돌파" 판정: 종가가 해당 추세선을 상향 통과한 봉이 1개 이상 확인된 경우.
- "지지" 판정: 추세선 ±2% 이내에서 반등이 확인된 경우.
- 5단계 "상승추세선 돌파"는 상승채널 **상단 평행선** 돌파 기준 (하단 상승추세선이 아닌 채널 천장 돌파).

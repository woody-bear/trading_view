# Quickstart: 차트 추세선 채널 검증 시나리오 (033-chart-trendlines)

## 사전 조건

- 백엔드: `uvicorn app:app --reload --host 0.0.0.0 --port 8000`
- 프론트엔드: `pnpm build && pnpm dev` (또는 `pnpm dev` 개발 중)
- 60일 이상 거래 이력이 있는 종목 (예: 삼성전자 005930, AAPL, BTC-USD)

---

## 시나리오 1 — 추세선 4선 표시 확인 (SC-001)

1. 종목 상세 화면 진입 (예: 삼성전자)
2. 차트 로드 후 3초 이내에 추세선 표시 확인
3. **검증**: 하락채널(빨간 실선 + 빨간 점선 = 2선), 상승채널(초록 실선 + 초록 점선 = 2선) 총 4선
4. **Pass**: 4선 모두 캔들 차트 위에 오버레이 표시됨

---

## 시나리오 2 — 기간 탭 전환 (SC-001, FR-001/002)

1. 기본 탭 "3M" 상태에서 차트 확인
2. "1M" 탭 클릭 → **즉시** 차트 범위 축소 + 추세선 재계산 확인
3. "6M" 탭 클릭 → 범위 확장 + 추세선 재계산 확인
4. "12M" 탭 클릭 → 최대 범위 + 추세선 확인
5. **검증**: 탭 전환 시 네트워크 탭(브라우저 DevTools)에서 새 API 요청 없음
6. **Pass**: 탭 전환 1초 이내 완료, 재요청 없음

---

## 시나리오 3 — 추세 단계 패널 표시 (SC-002)

1. 종목 상세 화면에서 차트 아래로 스크롤
2. "추세 전환 단계" 패널 확인
3. **검증**: 5개 단계 항목 표시, 완료 단계는 체크마크(✓), 현재 진행 단계는 강조
4. **Pass**: 단계 번호(0/5 ~ 5/5) 정확히 표시

---

## 시나리오 4 — 거래량 하이라이트 (FR-010/011)

1. 단계 완료가 있는 종목 확인 (단계 1 이상 완료)
2. 차트의 볼륨 바 확인 — 단계 전환 시점 봉의 볼륨 바가 노란색으로 표시
3. 패널의 완료 단계에 "거래량 X.X×" 수치 표시 확인
4. **Pass**: 볼륨 바 시각적 구분 가능, 패널 수치 존재

---

## 시나리오 5 — 추세선 가격 수치 표시 (SC-003)

1. 패널에서 각 단계의 추세선 가격 확인 (예: "하락추세선 72,300원")
2. 차트에서 해당 추세선의 현재 가격 위치 육안 확인
3. **Pass**: 패널 수치와 차트 시각 위치 대략 일치 (±5% 허용)

---

## 시나리오 6 — 데이터 부족 종목 (SC-005, FR-007)

1. 신규 상장 종목 또는 60일 미만 데이터 종목 접근
2. **검증**: 차트는 정상 표시, 추세선 없음, 패널에 "분석 불가 — 데이터 부족" 메시지
3. **Pass**: 오류(빨간 에러 화면) 없이 안내 텍스트만 표시

---

## 시나리오 7 — 모바일 레이아웃 (SC-004)

1. 브라우저 DevTools에서 모바일 에뮬레이션 (375px 이하)
2. 차트 추세선 표시 확인
3. 단계 패널이 스크롤 가능한 카드 형태로 표시 확인
4. **Pass**: 레이아웃 깨짐 없음

---

## 회귀 확인

- 기존 차트 BUY/SELL 마커 정상 표시 (추세선과 독립)
- 기존 TrendAnalysisCard (024) 정상 표시
- 스캔 페이지 BUY 종목 목록 정상 표시
- 기간 탭 선택 없이 차트 접근 시 기본 3M으로 표시
